# JOJIFrontier 仲間・加入設計

## Canon status

加入順、兵種、ゲーム内条件はJOJIFrontier固有仕様として確定する。新規人物名と経歴は
World Bibleへ未登録の**Frontier提案**であり、共有Canonとしては未確定とする。

参照した正本:

- `../../JojiWorldBible/naming.md`
- `../../JojiWorldBible/characters.md`
- `../../JojiWorldBible/docs/world/embermarch.md`

Embermarchで既に確定しているRhaest Bloodmarch、Dael Bloodmarch、Ser Idris Ironwatch、
Soren Scorchreachは王家・軍上層部・著名な斥候であるため、通常の遠征隊員として流用しない。

12人のFrontier正式表示名、人物像、相互関係は[`cast_reference.md`](cast_reference.md)を正本とする。
World Bible登録までは共有Canon上の提案である。セーブ互換性のため、現行の`UnitId`は変更せず、
表示名だけを移行する。

## ロスター規模

- 基本ロスターは12人
- 1兵種につき基準となる名前付き人物1人
- 遠征は常に4人編成
- 同一人物の兵種変更は採用しない
- 仲間は死亡せず、HP0では遠征中の戦闘不能になる
- 拠点帰還でHPと戦闘不能を完全回復する
- 加入済み人物、装備、スキル構成は恒久保存する
- 施設建設だけで人物が自動加入することはない

## 加入処理の共通ルール

```text
探索または副目標を達成
-> 対応する人物を「加入候補」として記録
-> その戦闘に勝利
-> 人物が生存した状態でキャンプへ到達
-> 拠点へ安全帰還して「加入可能候補」として恒久化
-> 集会所で加入会話
-> rosterへ恒久追加
```

- 戦闘勝利だけでは恒久加入しない
- 敗北すると、その遠征中に得た加入候補は失う
- 同じ加入イベントは次回遠征で再挑戦でき、永久に加入不能にならない
- 加入候補を保護する護衛戦では、対象がHP0になっても死亡せず撤退扱いにする
- 撤退した場合、その遠征では加入せず、次回イベントを再生成する
- 宿舎の受け入れ枠が不足していても加入可能候補は消さず、集会所で枠不足を通知する
- 受け入れ枠を空けた時点で加入会話を再開できる
- 安全帰還後の加入可能候補は、その後の別遠征で敗北しても失わない
- 加入会話の選択肢で人物を永久拒否できない。保留した場合は集会所へ残る

### 受け入れ枠

| 状態 | 上限 | 解放条件 |
|---|---:|---|
| 共同テント | 6人 | ゲーム開始。初期6人全員を収容 |
| 宿舎増築I | 8人 | 灰枝の森攻略、安全帰還、開拓拠点化。素材消費なし |
| 専門区画 | 11人 | 沈黙した監視所群で野戦工作記録を安全帰還。素材消費なし |
| 遠征別棟 | 12人 | 灰鉄採石場攻略、異常鉱脈記録を安全帰還。素材消費なし |

- 受け入れ枠は基礎設備の進行であり、任意施設の建設を要求しない
- 兵種を早期に使えることが本作の中心なので、受け入れ枠に通常素材費を課さない
- 上限は加入済みRosterだけを数え、未加入候補を数えない
- Rosterから仲間を解雇する機能は採用しない
- 12人到達後も4人編成を維持する

## 加入ペース

- 最初から6兵種を比較可能にする
- 灰枝の森後に重装兵と辺境猟兵を加え、8兵種にする
- シンダーウォッチ方面への複数回の安全帰還を通じて工兵、伝令騎兵、旗手を加え、11兵種にする
- 灰鉄採石場攻略後に戦闘魔導士を加え、全12兵種を編成可能にする
- 第4地域以降は新兵種の解放ではなく、12兵種の組み合わせを試す地域にする
- 新加入直後の次の遠征には、その兵種専用の探索選択肢か副目標を最低1つ置く
- 早期加入者へ強力な分岐武器や第2・第3スキルを同時支給しない

## 早期兵種解放と成長の分離

JOJIFrontierの中心は「どの兵種4人を連れていくか」であるため、兵種そのものを長期間
ロックしない。灰鉄採石場攻略時点で全12兵種を揃える。

加入時に使用可能:

- 兵種固有能力
- 基本武器1本
- 各兵種の第1装備スキル
- 兵種固有の探索選択肢

加入後も段階解放するもの:

- 第2・第3装備スキル
- 3方向の武器分岐
- 調整特性
- 上位の施設効果
- 高品質消耗品と戦闘工作アイテム

兵種の選択肢は早く渡し、同じ兵種をどう調整するかは地域攻略と拠点発展で増やす。

## 人物設定の作成規則

各人物の正式設定には次の項目を必須とする。

| 項目 | 内容 |
|---|---|
| Canon名 | World BibleのEmbermarch命名規則に登録済みの名前 |
| 出身 | Embermarch内の既存地点、またはBibleへ追加承認された地点 |
| 立場 | 軍人、開拓民、暁の教会の衛生兵、外来者など |
| 遠征参加理由 | なぜ前線防衛線の本隊ではなく開拓遠征へ参加するか |
| 専門性 | 戦闘能力と探索能力が同じ経歴から説明できること |
| 個人的目的 | 地域攻略と別に、本人が遠征を続ける理由 |
| 対立 | 他の仲間と判断が分かれる具体的な論点 |
| 禁止事項 | 王族、エンバーマーチ統治者、前線防衛線総司令官など物語規模を壊す立場にしない |

### 性格の役割分担

- 行軍隊長: 判断を下すが、常に正解を知っている人物にはしない
- 古参守備兵: 慎重さを担当するが、変化を拒むだけの人物にはしない
- 監視弓兵: 情報を重視するが、斥候の探索能力を代行しない
- 暁の衛生兵: 帰還判断と人命を重視するが、遠征そのものへ反対し続けない
- 辺境斥候: 未知への好奇心を持つが、無謀さだけで問題を起こさない
- 槍兵: 規律と現場感覚を持つが、古参守備兵の若い複製にしない
- 重装兵: 正面突破を好むが、考えない怪力役にしない
- 辺境工兵: 実用性を優先するが、全問題を装置で解決しない
- 伝令騎兵: 迅速さを重視するが、斥候より地形知識に詳しくしない
- 辺境猟兵: 生態と追跡を理解するが、野生生物を悪と決めつけない
- 旗手: 集団の士気を扱うが、行軍隊長より上位の指揮官にしない
- 戦闘魔導士: 魔法知識を持つ外来の希少人物とし、世界の謎をすべて説明できる存在にしない

## UI・セーブ

- 未加入人物は宿舎にシルエットや名前を表示しない
- 加入候補になった人物だけ「帰還後に加入可能」とキャンプで表示する
- 加入時に兵種、固有能力、探索能力、初期武器を1画面で表示する
- 初期装備スキルは各兵種の第1スキルのみ解放済みとする
- 加入人物には対応する基本武器を1本支給する
- `UnitId`、加入状態、装備、スキルを保存し、表示名はデータ定義から読み直す
- Schema移行では旧表示名を保存キーに使わない
- 加入可能候補IDと加入済みUnit IDを別の集合として保存する
- 集会所は加入会話、宿舎はRoster・4人編成、ユニットページは装備とスキルを担当する

## 兵種加入時の付与

加入確定時に一度だけ次を実行する。

1. Unit IDをRosterへ追加
2. 対応する基本武器IDを恒久装備候補へ追加
3. 兵種固有能力と探索能力を有効化
4. `skill_system.md`のTier 1スキルを解放して第1枠へ装備
5. 第2スキル枠を空で初期化
6. 初期装備、兵種役割、探索能力を加入結果画面へ表示
7. 恒久セーブを実行

同じ加入会話や帰還処理を再実行しても、武器、スキル、Rosterを二重追加しない。

## 確定テスト

1. ゲーム開始時に初期6人を編成候補として使用できる
2. 灰枝の森帰還後に受け入れ上限8へ上がる
3. 沈黙した監視所群の途中で上限11となり、工兵・騎兵・旗手を受け入れられる
4. 灰鉄採石場帰還後に上限12となり、全12兵種を使用できる
5. 敗北した遠征の加入候補は失い、次回再挑戦できる
6. 安全帰還済みの加入可能候補は別遠征の敗北で失わない
7. 集会所会話前にはRosterへ追加されない
8. 加入処理を再実行してもRoster、基本武器、Tier 1スキルが重複しない
9. 未加入人物の名前と能力を通常の宿舎一覧へ表示しない
10. 加入会話の保留で人物を永久に失わない
11. 全12兵種が灰鉄採石場攻略時点で使用可能になる
12. 加入済み人物のUnit ID、装備、スキルがセーブ後に復元される
