# JOJIFrontier 状態異常・一時効果

## 状態

共通ゲーム仕様と初期値は確定。データ基盤・5種類の効果処理・地形処理順を実装済み(`jf/core/StatusEffect.hpp`、
`jf/battle/StatusEffects.hpp`)。毒、炎上、移動低下、防御低下、よろめきの5種類に限定する。
煙幕と炎上床はユニット状態ではなくマス効果として別管理し、まだ実装されていない
（対応するTerrainTypeが未追加）。

状態異常は本書の5種類だけを指す。挑発、標的指定、隊形維持などは`skill_system.md`の
一時戦術効果として別管理し、万能薬や状態治療では解除しない。

実装済み:

- `StatusEffectType`5種と`Unit`側の残り発動回数・持続フラグ、ボス補正値
  (`statusPoisonDamage()`などisBoss別の関数)
- `applyPoison()`/`applyBurn()`/`applyMoveDown()`/`applyDefenseDown()`/`applyStagger()`:
  同名重複なし、再付与は規定値へ戻す、よろめきは免疫中なら無効
- `processActionEndStatusEffects()`: 行動終了時の炎上ダメージとよろめき解除
  (`BattleController::finishPlayerAction()`と`EnemyAI`の両方から呼び出し)
- `processPhaseEndStatusEffects()`: Phase終了時の毒ダメージ、移動低下／防御低下／
  よろめき無効の解除（`BattleController`のPlayer/Enemy Phase終了処理から呼び出し）
- `clearAllStatusEffects()`: Unit単位（万能薬・状態治療用、よろめき免疫は対象外）と
  BattleState単位（戦闘終了時、免疫も含めて全解除）の2形態
- `Unit::effectiveMove()`/`effectiveDefense()`: 実際の移動力・防御力算出に反映
  (`Movement.cpp`の到達マス計算、`CombatResolver.cpp`のダメージ計算)

実際に付与する経路(2026-07、M4項目4で追加):

- 移動低下: 監視弓兵`suppressing_shot`(制圧射撃)・槍兵`halting_thrust`(足止め突き)が命中時に
  `applyMoveDown()`を呼ぶ(`BattleController.cpp`の`attackSkillShapes`テーブル、
  `appliesMoveDown`パラメータ)
- よろめき: `BattleState::applyKnockback()`が、ノックバック先が範囲外・他Unit占有・不可通行
  地形・Battle Object(倒木などのBarrier)のいずれかで塞がれている場合に`applyStagger()`を
  呼ぶ(元々このBattle Objectチェック自体が存在せず、ノックバックがBarrierを無視して
  素通りできてしまうバグも同時に修正した)
- 武器共通: `Weapon::onHitStatuses`を追加し、通常攻撃・反応攻撃・警戒射撃を含む
  `resolveAttack()`経路で、命中時に5種類の任意の組合せを確定付与する。`weapons.json`では
  安定文字列IDで指定する。現行装備の数値・追加効果は変更せず、地域別装備追加時に利用する
- 防御低下: 辺境斥候`ambush`(奇襲)が命中時に付与する
- 浅瀬: 行動終了時、炎上ダメージより先に炎上を解除する

未実装（今後の課題）:

- 毒を付与する正式な現行装備・敵。共通の武器付与経路は実装済み
- 炎上: 「投擲火炎壺、火系魔法、炎上床」が主な発生源だが、対応するアイテム・スキル・地形が
  未実装。武器共通の炎上付与経路自体は実装済み
- 工兵固有の防御低下Skillは未実装。「倒木衝突」経由の防御低下はBoss専用の別ID
  (`bossWeakenedFromStun`)で既に実装済み
  (本書「ボス補正」の通り、通常のdefenseDownActiveとは意図的に別管理)
- 万能薬・状態治療スキル自体（`clearAllStatusEffects(Unit&)`は関数として存在するのみ。
  暁の衛生兵の装備Skill`cleanse`は既にこれを呼んでいる）
- 煙幕・炎上床のマス効果、ボスユニット自体（`Unit::isBoss`はフラグとして存在するのみ）

## 共通ルール

- 付与確率は使わず、攻撃またはスキルが命中すれば確定付与
- 同名効果は重複しない
- 再付与時は効果量を加算せず、規定の残り時間へ戻す
- 異なる状態異常は同時に付与可能
- 状態異常は戦闘終了時にすべて解除し、次の戦闘へ持ち越さない
- 状態異常によるHP0は通常の戦闘不能として処理する
- 毒だけはHPを1未満にしない
- 万能薬は対象1人の全状態異常を解除する
- 暁の衛生兵の状態治療スキルは対象1人の全状態異常を解除する
- HP回復だけでは状態異常を解除しない
- 敵味方で同じ効果定義と持続時間を使用する

持続時間の減少規則は `skill_system.md`に従う。

## 初期5種類

### 毒

| 項目 | 仕様 |
|---|---|
| 効果 | 対象側Phase終了時に2ダメージ |
| 持続 | 3回発動すると解除 |
| 戦闘不能 | HP1未満にはならない |
| 再付与 | 残り発動回数を3へ戻す |
| 無効対象 | ゴーレム、構造物、召喚物など明示された無機物 |
| 主な発生源 | 獣・魔物の攻撃、毒罠、将来の消耗品 |

役割は長期戦への圧力。即時火力や移動制御は持たせない。

### 炎上

| 項目 | 仕様 |
|---|---|
| 効果 | 対象が行動終了したとき3ダメージ |
| 持続 | 2回発動すると解除 |
| 戦闘不能 | HP0になり得る |
| 再付与 | 残り発動回数を2へ戻す |
| 地形解除 | 浅瀬で行動終了すると、ダメージ発生前に解除 |
| 主な発生源 | 投擲火炎壺、火系魔法、炎上床 |

隣接ユニットや隣接タイルへ自動拡散しない。役割は行動を急がせ、安全な解除地点を選ばせること。

### 移動低下

| 項目 | 仕様 |
|---|---|
| 効果 | MOV-2、最低MOVは1 |
| 持続 | 対象側の次Phase終了まで |
| 再付与 | 持続時間だけ更新 |
| 主な発生源 | 制圧弓、鉄杭、罠、泥濘系ギミック |

役割は敵経路の制御。攻撃・スキル・アイテムの使用は妨害しない。

### 防御低下

| 項目 | 仕様 |
|---|---|
| 効果 | DEF-3 |
| 持続 | 対象側の次Phase終了まで |
| 再付与 | 持続時間だけ更新 |
| RES | 低下しない |
| 主な発生源 | 倒木衝突、重装破壊、武器・工兵スキル |

役割は高DEF対象へ集中攻撃の機会を作ること。魔法防御低下は初期実装に含めない。

### よろめき

| 項目 | 仕様 |
|---|---|
| 効果 | 対象の次の行動では移動不可 |
| 可能な行動 | 現在地から攻撃、スキル、アイテム、待機 |
| 解除 | 対象が行動終了すると解除 |
| 再付与耐性 | 解除後、対象側の次Phase終了までよろめき無効 |
| 主な発生源 | 障害物へのノックバック衝突、閃光筒、強い迎撃 |

完全な行動不能は初期状態異常として採用しない。役割は位置を固定しつつ、最低1つの行動を
残すこと。

## ボス補正

Bossの予告行動、強制移動、Phase移行、退場理由は
[`boss_common_rules.md`](boss_common_rules.md)を参照する。

ボスへ通常状態異常を完全な効果量で適用しない。個別の完全無効がある場合は戦闘前に表示する。

| 状態 | 通常ボスへの補正 |
|---|---|
| 毒 | Phase終了時1ダメージ、最大2回 |
| 炎上 | 行動終了時2ダメージ、最大2回 |
| 移動低下 | MOV-1 |
| 防御低下 | DEF-2 |
| よろめき | 移動不可ではなく、次の行動だけMOV-1 |

ボス専用ギミックによるスタン、弱体、行動不能は通常のよろめきと別IDで管理する。灰角大猪の
倒木衝突など、攻略手順として設計された効果には専用ルールを適用する。

## 地形・マス効果との関係

| 地形・効果 | 状態異常との関係 |
|---|---|
| 浅瀬 | 行動終了時、炎上ダメージより先に炎上を解除 |
| 茂み | 状態異常耐性を与えない |
| 薬草地点 | HP5回復のみ。状態異常は解除しない |
| 煙幕 | マス効果。中にいる対象への攻撃・視認ルールを変更する |
| 炎上床 | マス効果。行動終了時に炎上を付与する |
| 倒木衝突 | 防御低下またはボス専用スタンを付与する |

地形効果の処理順:

1. 行動終了地点を確定
2. 浅瀬など解除効果
3. 薬草地点など回復効果
4. 炎上床など状態付与
5. ユニット状態の行動終了時ダメージ
6. 戦闘不能判定

この順序により、炎上したユニットが浅瀬へ到達した場合は炎上ダメージを受けずに解除できる。

## UI

- ユニット円の近くへ状態アイコンを最大3個表示
- 4個以上ある場合は3個目を`+N`表示にする
- アイコンへカーソルを合わせると効果量、残り時間、解除方法を表示
- ユニット詳細に全状態を一覧表示
- 戦闘予測に攻撃後付与される状態を表示
- 完全無効、ボス補正、再付与耐性を対象選択時に表示
- Phase開始・終了時ダメージはHP減少と状態名を短時間表示
- 日本語名は「毒」「炎上」「移動低下」「防御低下」「よろめき」で統一

### 実装状況

実装済み(`src/main.cpp`の`activeStatusBadges()`/`drawGrid()`/`drawHoverInfo()`):

- 戦闘マップのユニット円下、HPバーの下へ最大3個の丸バッジを表示。4個以上は3個目が
  `+N`に切り替わる
- ホバーツールチップに、アクティブな状態ごとの日本語名＋残り回数／解除条件を1行ずつ表示
- ツールチップのSTR/MAG/DEF/RES/MOV行は`effectiveDefense()`/`effectiveMove()`を表示し、
  防御低下・移動低下・よろめきを反映した実数値を見せる

未実装:

- ユニット詳細画面での全状態一覧 - 拠点のユニットページ(`drawUnitScreen`)は`UnitTemplate`
  （拠点ロースター）を扱い、状態異常を持つ`Unit`（戦闘中インスタンス）には対応する画面が
  ない。ホバーツールチップが現状唯一のin-battle詳細表示
- 戦闘予測(`CombatPreview`)への「攻撃後付与される状態」表示 - 現状どの攻撃・スキルも

## 敵AI

- 状態異常で撃破できるかを通常攻撃の撃破判定と区別する
- 毒状態の敵はHP1で残ることを理解する
- 炎上した敵は到達可能なら浅瀬を解除候補として評価する
- 移動低下中は到達可能範囲を再計算する
- 防御低下中の対象は集中攻撃候補として評価する
- よろめき中は移動計画を作らず、現在地から可能な行動を選ぶ
- 再付与しても効果が増えない状態へ、無意味な重複スキルを使わない
- ボス補正後の実際の効果量を評価する

## 実装時の不変条件

- 状態異常付与へランダム確率を追加しない
- 初期実装で5種類を超えない
- 毒と炎上を同じ持続ダメージにしない
- 移動低下とよろめきを同じ移動妨害にしない
- 状態異常だけで通常ボスを永久停止できない
- 解除手段なしの状態をプレイヤーへ恒常的に使用しない
- 状態異常を遠征間の新しい恒久消耗へしない

## 確定テスト

1. 同名状態を再付与しても効果量が増えず、残り時間だけ規定値へ戻る
2. 毒はPhase終了時に発動し、HP1で止まる
3. 炎上は行動終了時に発動し、HP0になり得る
4. 浅瀬では炎上ダメージより先に解除する
5. 移動低下後も最低MOV1を維持する
6. 防御低下はDEFだけを下げ、RESを変更しない
7. よろめき中は移動不可だが現在地から行動できる
8. よろめき解除後の耐性中は再付与できない
9. ボスへ補正後の効果量を使用する
10. 戦闘終了時に全状態とよろめき耐性を解除する
11. 状態異常による戦闘不能でも`UnitDefeatedEvent`を1回だけ発行する
12. 万能薬と状態治療は5状態を解除するが、一時戦術効果と任務フラグを解除しない
