# JOJIFrontier 拠点発展システム

状態: JOJIFrontier固有の共通ゲーム仕様として確定。地域名や世界設定へ接続する際は
`../JojiWorldBible` を確認して正史との整合を取る。

遠征成果のPending分類と安全帰還は[`expedition_rewards.md`](expedition_rewards.md)を正本とする。
全62地点の取得物と施設ノードの対応は
[`stage_facility_progression.md`](stage_facility_progression.md)を正本とする。
全施設研究ノードのID、効果、条件、費用は
[`facility_research.md`](facility_research.md)を正本とする。

## 基本方針

拠点は施設レベル制ではなく、開拓段階と分岐ノードで発展する。重要地点の
確保は拠点全体の発展上限を開くが、個々の技術を自動解放はしない。

基本循環:

1. 遠征で知識、人物、施設、素材を発見する。
2. 無事に帰還し、未確定の発見物と戦利品を恒久資産にする。
3. 前提発見と素材を揃えて施設ノードを解放する。
4. 新しい戦闘、探索、編成、装備の選択肢を次の遠征で使う。

帰還前の発見物と素材は敗北時に失う。解放済みノードは恒久で失われない。

## 拠点全体の開拓段階

| 段階 | 名称 | 主な範囲 |
|---|---|---|
| 0 | 野営地 | 初期6兵種、基本物資、焚火場での会話 |
| 1 | 開拓拠点 | 各施設の第1分岐、地域情報、共同食堂 |
| 2 | 辺境集落 | 全12兵種、専門装備、探索道具、集会所 |
| 3 | 開拓都市 | 上位分岐、深層遠征、特殊遠征、開拓会館 |

段階はノードへのアクセス上限であり、技術そのものではない。

### 施設利用

作戦テント、倉庫、共同テント、集会所はゲームの基本ループに必要な基礎設備として常時利用できる。

救護テント、訓練場、工作台、簡易鍛冶台は、初回建設後は恒久的に拠点へ残り、常時利用できる。
同時稼働枠、停止状態、遠征前の稼働切替は採用しない。素材が足りれば4施設すべてを順次建設できる。

### 建設と研究

- `unlockedNodeIds`は知識と研究の恒久記録
- `constructedFacilityIds`は初回建設済みの物理施設を表し、削除しない
- 初回建設時だけ建設費を払い、建設済み集合へ追加する
- 建設済み施設は常時利用可能で、稼働・停止状態を持たない
- 解体、素材返却、再建費は採用しない
- 製作済み武器・特性・消耗品・探索道具は施設状態によって失われない

### 施設状態の正式な分離

施設と技術の状態は次の4種類を混在させない。

| 状態 | 意味 | 失われる条件 |
|---|---|---|
| 未建設／建設済み | 物理施設が存在するか | 建設済み施設は失われない |
| 未研究／研究済み | Discoveryと研究費で技術を習得したか | 研究済み技術は失われない |
| 未所有／所有済み | 武器、特性、道具を製作して所持しているか | 消耗品の使用時だけ個数が減る |

- 施設に`Lv`は設定しない。UIは開拓段階、建設状態、研究済み分岐数を表示する
- 研究済み分岐はすべて恒久的に利用可能で、分岐同士の排他選択や再研究を要求しない
- 戦略上の制約は、4人編成、武器1枠、スキル2枠、調整特性1枠、消耗品6枠、探索道具2枠で作る
- 建設済み施設を利用できない状態が発生した場合は実装不具合とする

旧Schemaの`builtNodeIds`は移行時に`constructedFacilityIds`へコピーする。研究IDは
`unlockedNodeIds`へそのまま保持し、旧稼働状態は読み捨てる。

### 建設後に常時可能な操作

- 製作済み武器の装備変更
- 解放済みスキルの装備変更
- 所有済み調整特性の付け替え
- 所有済み消耗品6枠と探索道具2枠の準備
- 4人編成、ユニット閲覧、倉庫、集会所、セーブ

### 建設済み施設で可能になる操作

| 施設 | 可能になる操作 |
|---|---|
| 診療所 | 医療研究、医療消耗品の新規製作 |
| 訓練所 | 新スキル研究、解放条件の確定 |
| 工房 | 戦闘道具・探索道具の新規製作、工作研究 |
| 鍛冶場 | 武器・調整特性の新規製作、鍛造研究 |

既に所有・解放済みのものを遠征へ装備する操作にも施設の再選択を要求しない。

### 進行不能防止

- 遠征先選択、倉庫、4人編成、ユニット閲覧、集会所、セーブは基礎設備として常時利用可能
- 次地域の解放、仲間の正式加入、拠点段階上昇を施設建設条件にしない
- 重要発見とキー素材は施設建設、武器製作、消耗品製作で消費しない
- 拠点段階上昇で消費できるキー素材は、その段階上昇だけに用途を限定する
- 必須の建設素材は対応地域の主目的と安全帰還だけで最低量を確保できるようにする
- 副目標、兵種専用探索、高難度ルートを必須発展素材の唯一の取得方法にしない

## 共通の解放条件

全ノードの条件は次の分類で統一する。

- 必要開拓段階
- 持ち帰った発見、記録、設計図
- キー素材と一般素材
- 前提施設ノード
- 必要に応じて専門家の加入

施設効果は原則として新しい行動、ルート、アイテム、編成判断を最低1つ追加する。
単純な数値上昇は副次効果に留める。

条件評価順は、開拓段階、重要発見、専門家、前提ノード、素材の順とする。UIは最初に
満たしていない条件だけでなく、すべての条件と現在値を表示する。素材消費は全条件成立後に一括で
行い、途中まで消費して失敗する処理を許可しない。

## 1. 司令所

初期ノード「作戦テント」:

- 遠征先を選択
- 4人編成を確認
- 基本的な危険度を表示

分岐:

- 偵察網: 敵の種類、敵の初期配置を表示
- 地図室: ルート分岐の一部、ボスまでのおおよその距離を表示
- 遠征計画室: 深層情報を表示し、特殊遠征を解放

## 2. 訓練所

全12人の加入時装備、連携作戦、調整特性、ユニットページは
[`character_progression.md`](character_progression.md)を正本とする。

初期ノード「訓練場」: 基礎兵種と基本兵種技を使用可能にする。

分岐:

- 前衛訓練: 古参兵、槍兵、重装兵技術
- 機動訓練: 斥候、猟兵、伝令騎兵技術
- 専門訓練: 工兵、旗手、上位兵種分岐

兵種解放は通常レベル成長ではなく、盤面能力と探索選択肢を増やす。

## 3. 鍛冶場

初期ノード「簡易鍛冶台」:

- 武器・調整特性の製作と解放
- 武器ごとに調整特性を1つ装着
- 製作済み装備の変更は拠点のユニットページで行う
- 装備変更は遠征前のみ可能
- 武器分岐レシピの閲覧と製作
- Iron系の基本武器を使用可能

装備は兵種単位ではなくユニット単位で保存する。鍛冶場ページはレシピ、必要素材、製作・解放を
扱い、ユニットページはステータス、兵種、武器、調整特性、将来のスキル装備を扱う。

通常の使用回数制武器耐久は採用しない。武器は通常使用で破損せず、遠征中の修理操作も
行わない。鍛冶場は性能を元へ戻す施設ではなく、戦い方を変更する施設とする。

分岐:

- 武器鍛造: 高品質武器、武器固有特性
- 重装加工: 防具強化、ノックバック耐性装備
- 特殊素材加工: 魔物素材武器、遺跡素材武器

武器は `+1` から `+10` ではなく機能別に分岐する。例としてIron Spearは、
射程型Long Spear、ノックバック型Heavy Spear、迎撃型Guard Spearへ分かれる。

成長順序:

```text
基本武器
-> 小規模調整（特性1枠）
-> 新素材・重要発見・鍛冶技術を獲得
-> 武器分岐
-> 数値上位ではなく戦い方が変化
```

最初の調整特性:

| 特性 | 費用 | 効果 |
|---|---|---|
| 獣皮の柄巻き | 獣皮1 | 戦闘ごとに最初に受けるノックバックを1回無効化 |

最初は獣皮の柄巻きだけを使用可能にし、調整システムを一度に増やしすぎない。

将来のIron Spear分岐:

- Long Spear: 射程型。最大射程+1、威力-2
- Heavy Spear: 火力型。威力+2、MOV-1
- Guard Spear: 迎撃型。迎撃姿勢強化、通常威力-1

### レシピ解放の段階

18本を一度に解放しない。地域ごとに各兵種へ選択肢を少しずつ追加する。

| 拠点段階 | 解放範囲 | 目的 |
|---|---|---|
| 野営地 | 基本武器のみ | 兵種固有能力を先に理解する |
| 開拓拠点 | 各初期兵種の第1分岐、槍兵3分岐 | 最初の横方向カスタマイズを試す |
| 辺境集落 | 各兵種の第2分岐 | 高火力など明確な代償を持つ型を追加 |
| 開拓都市 | 各兵種の第3分岐 | 再移動、弱体、範囲支援など複雑な型を追加 |

製作費は地域固有素材を消費するが、地域進行に必須の重要発見やボスキー素材そのものは
消費しない。重要発見はレシピの知識条件、木材・鉄材・獣皮・薬草などは製作費として分離する。
初期6兵種の各レシピに必要な素材数とDiscoveryは
[`character_progression.md`](character_progression.md#初期6兵種の武器レシピ)を正本とする。

### 実装時の検証基準

- 基本武器を選ぶ価値が残ること
- 1つの分岐が同兵種の他2分岐を常に上回らないこと
- MOV増加、再移動、射程4は同時に同じユニットへ付与しないこと
- 足止めや弱体はボスへ無効または効果量半減とし、専用AIを破壊しないこと
- Heal追加効果はアイテム回復へ適用しないこと
- 装備変更後の戦闘プレビューへ威力、射程、MOV、固有効果を反映すること

灰角の大牙は開拓拠点への発展に使うキー素材であり、武器製作には使わない。将来、灰角系の
武器調整を追加する場合は別素材「灰角の欠片」を使用し、必須進行素材と競合させない。

### 後半武器の素材・解放段階

| 段階 | 主な解放 | 使用素材 |
|---|---|---|
| 開拓拠点 | 後半兵種の基本武器だけ | 加入時支給。製作不要 |
| 辺境集落 | 各兵種の第1分岐 | 木材、鉄材、獣皮、薬草 |
| 開拓都市 | 各兵種の第2・第3分岐 | 通常素材と対応する重要発見 |
| 魔導工房解放後 | 戦闘魔導士の分岐 | 遺跡片と魔法の重要発見 |

- `遺跡片`は通常戦利品であり、重要発見や地域ボスのキー素材ではない
- 基本武器は加入時に1本支給し、紛失・耐久・修理を採用しない
- 製作済み武器は恒久在庫へ入り、同じ武器を再製作する必要はない
- 素材が不足する場合は製作不可理由と不足数を鍛冶場UIへ表示する
- 武器分岐は兵種を別兵種へ変えず、`class_reference.md`の役割境界を越えない
- 固有効果の状態異常は`status_effects.md`、再移動と一時強化は`skill_system.md`に従う

## 4. 診療所

初期ノード「救護テント」: 救急セットと救命包を解放する。

分岐:

- 薬学: 万能薬、特殊状態異常治療

拠点帰還時は常に完全回復する。診療所は拠点回復ではなく、遠征中の医療能力を
強化する施設とする。

## 6. 宿舎

初期ノード「共同テント」: 初期仲間6人と4人編成を使用可能にする。

分岐:

- 宿舎増築I: 灰枝の森攻略・安全帰還・開拓拠点化後。重装兵と辺境猟兵を含む8人まで受け入れ
- 専門区画: 沈黙した監視所群で野戦工作記録を安全帰還。工兵、伝令騎兵、旗手を含む11人まで受け入れ
- 遠征別棟: 灰鉄採石場攻略・異常鉱脈記録を安全帰還。戦闘魔導士を含む12人を受け入れ、全兵種を編成可能にする
- 交流区画: 仲間イベント、連携能力

施設解放は受け入れを可能にするだけで、仲間は探索や地域イベントを通じて加入する。
宿舎は常設の基礎設備である。3つの宿舎拡張は基礎進行として素材費なしで解放する。
基本ロスターは1兵種につき1人なので、現行12人では同一兵種制限を設けない。将来同じ兵種の
人物を追加する場合は同一兵種2人までとし、追加補給コストは課さない。

## 7. 集会所

集会所は人の話を聞き、加入、地域証言、施設要望、遠征情報を確認する常設基礎設備である。
常設の基礎設備で、拠点段階に応じて焚火場、共同食堂、集会所、開拓会館へ外観が変化する。

扱うもの:

- 仲間の加入会話
- 仲間同士の意見と地域総括
- 開拓民、工人、衛生兵、伝令、来訪者の証言
- 新しい施設ノードの存在と必要条件
- 次の遠征地に関する確定情報と噂
- 攻略後に生活と拠点がどう変わったか

扱わないもの:

- 装備変更、スキル変更、アイテム製作
- 数値好感度、贈り物、恋愛
- 会話だけによる能力値上昇
- 任意会話未読による主進行停止

詳細は[`gathering_place.md`](gathering_place.md)を正本とする。宿舎は人物・編成・ユニット管理、
集会所は会話と加入確認を担当し、同じ画面へ統合しない。

`共同施設研究`は施設枠の代替名称ではない。旧辺境集落の工人・衛生兵・守備隊から得た作業記録を
各施設で共有し、第7地域以降の「複数施設を前提にする研究」を研究可能にする恒久条件である。

- 診療所と工房: 危険地形用の治療・保護道具
- 工房と鍛冶場: 耐熱加工、特殊素材加工、上位戦闘工作
- 訓練所と宿舎: 連携作戦と集団防衛技術
- 司令所と工房: 後半地域の補給・帰還設備

個別効果と製作費は後続地域のDiscoveryを得てから解放するため、共同施設研究だけで自動取得しない。
施設の同時稼働数、能力値、バッグ枠、装備枠は増加しない。

- 同じ素材の内部IDは地域をまたいで統一し、`鉄鉱石`、`鉄材`、`高品質鉄材`を別IDとして扱う
- 地域専用書は上表の到達時点までに、主目的だけで必要量を確保できる最低保証を記載する
- すでに素材を別用途へ使用して不足した場合は再調査で通常素材を再取得できる
- キー素材の灰角の大牙だけは再取得前提にせず、拠点発展以外へ使用できない

## 設定完了条件

共通拠点仕様、初期4施設、施設研究費は確定済み。施設ノードの費用は
[`facility_research.md`](facility_research.md)を正本とする。新しい地域固有ノードを
追加する場合だけ、地域専用書の最低保証報酬と同時に費用を確定し、未設定をデータ読込時に検出する。

実装時は次を自動テストする。

1. 灰枝の森攻略後、素材条件を満たせば初期4施設をすべて建設できる
2. 建設済み施設は常時、新規研究と製作に利用できる
3. 稼働・停止・施設枠の状態をSaveへ持たない
4. 建設済み施設、Discovery、研究、製作品が失われない
5. 所有済み装備・スキル・特性・消耗品・探索道具を常に準備できる
6. 重要発見とキー素材を通常製作で消費できない
7. 拠点段階上昇の素材消費が原子的で、失敗時に在庫が変化しない
8. どの施設が未建設でも次地域へ出発できる
9. 旧`builtNodeIds`を建設済み施設へ安全に移行し、旧稼働状態を読み捨てられる
10. 施設の建設状態で装備中の武器・スキル・特性が解除されない
11. 研究済み分岐に追加の有効化枠や再研究を要求しない
12. UIに施設Lvや稼働状態を表示せず、建設・研究・所有の3状態を混同しない
