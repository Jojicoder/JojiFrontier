# JOJIFrontier 集会所・会話システム

更新日: 2026-07-13

状態: JOJIFrontier固有の拠点会話システムとして確定。人物名と共有世界の経歴はWorld Bibleを
正本とし、本書は会話の発生条件、UI、ゲーム進行への接続だけを定義する。

参照したWorld Bible:

- `../../JojiWorldBible/docs/world/embermarch.md`
- `../../JojiWorldBible/culture.md`
- `../../JojiWorldBible/religion.md`
- `../../JojiWorldBible/characters.md`
- `../../JojiWorldBible/naming.md`

エンバーマーチは恒常的な軍事社会で、一般的な平時の祭礼や娯楽施設を中心にしない。そのため、
華やかな酒場ではなく、食事、報告、負傷者の消息、街道情報を交換する実務的な集会所とする。

## 拠点画面での位置

拠点の主要画面は次の順で並べる。

```text
施設
遠征
集会所
ユニット
設定
```

集会所は常設の基礎設備である。建設忘れによって仲間加入や物語進行が止まらないようにする。

## 発展段階ごとの姿

| 拠点段階 | 見た目上の場所 | 主な利用者 | 機能 |
|---|---|---|---|
| 野営地 | 焚火場 | 初期隊員、少数の開拓民 | 初期会話、遠征前後の短い会話 |
| 開拓拠点 | 共同食堂 | 仲間、工人、衛生兵、運搬人 | 加入会話、地域証言、施設要望 |
| 辺境集落 | 集会所 | 住民、商人、伝令、来訪者 | 地域別会話、対立会話、依頼候補 |
| 開拓都市 | 開拓会館 | 各施設代表、遠征帰還者、外来者 | 本編後半会話、地域総括、深層情報 |

UI上のボタン名は全段階で「集会所」とし、画面タイトルと背景だけを発展段階に合わせて変える。

## 目的

- 仲間を能力表ではなく人物として理解できる場所を作る
- 遠征で得た記録が、実際に暮らす人々へどう影響したか示す
- 次の遠征地について、確定情報と噂を区別して伝える
- 加入候補との正式な加入会話を行う
- 施設建設の意味を工人、衛生兵、兵士の言葉から説明する
- プレイヤーが会話を読みたい時にまとめて読めるようにする

集会所は戦闘能力を直接上げる施設ではない。会話を全回収しないと通常戦闘で不利になる設計は
採用しない。

## 会話の種類

| 種類 | 必須性 | 内容 | 主な効果 |
|---|---|---|---|
| 加入 | 進行上重要 | 加入候補が遠征隊へ参加する | rosterへ恒久追加 |
| 本編 | 進行上重要 | 地域攻略、信号、拠点発展の確認 | 次地域や重要画面の解放確認 |
| 仲間 | 任意 | 個人的目的、判断、他の仲間との意見差 | 人物理解、後続会話フラグ |
| 開拓民 | 任意 | 道、水、食料、仕事、負傷者、家族 | 地域攻略後の生活変化を表示 |
| 施設 | 任意 | 必要素材、施設の用途、次の強化候補 | 施設ノードの情報を可視化 |
| 遠征情報 | 任意 | 地形、敵、旧街道、天候の証言 | 情報欄または探索選択肢の予告 |
| 対立 | 任意 | 続行か帰還、開拓か保全などの意見差 | 選択履歴と人物関係の描写 |
| 地域総括 | 任意 | 攻略済み地域を複数人物が振り返る | 記録帳へ会話を保存 |

## 会話データ

```cpp
enum class ConversationKind {
    Recruitment,
    MainStory,
    Companion,
    Settler,
    Facility,
    ExpeditionIntel,
    Disagreement,
    RegionReview,
};

struct ConversationDefinition {
    std::string id;
    ConversationKind kind;
    std::vector<std::string> speakerIds;
    std::vector<ConversationCondition> conditions;
    std::vector<ConversationLine> lines;
    std::vector<ConversationChoice> choices;
    std::vector<ConversationEffect> effects;
    int priority = 0;
    bool repeatable = false;
};
```

- 会話IDはASCIIの`lower_snake_case`
- 表示名や本文を判定キーに使わない
- `speakerIds`は名前ではなく安定人物IDを使う
- 一般開拓民は`settler_carpenter_01`のような役割IDを持たせる
- 本文は日本語の言語データから取得する
- 会話本文をC++の条件分岐へ直接埋め込まない

## 発生条件

会話条件は次の組み合わせだけを初期実装で使用する。

- 拠点段階
- 到達済み・攻略済み地域
- 安全帰還回数
- 登録済み重要発見
- 建設済みまたは解放済み施設ノード
- 加入済み人物、加入候補
- 直前の遠征結果
- 特定Objectiveの達成
- 会話既読ID
- 会話選択フラグ

戦闘の乱数Seed、表示言語、現在の所持金額を会話解放条件にしない。素材不足を話題にする場合も、
条件は「必要量未満」のようなデータ判定とし、特定の偶然値へ依存させない。

## 会話効果

初期実装で許可する効果:

- 仲間の恒久加入
- 会話フラグの登録
- 地域情報の公開
- 探索選択肢の予告または追加
- 施設ノードの存在と必要条件を公開
- 記録帳への追加

会話だけで許可しない効果:

- STR、DEF、MOVなどの恒久上昇
- 武器や高品質素材の無条件配布
- 重要発見の代替取得
- Pending資産の安全帰還前確定
- 読まなかった会話による人物永久ロスト
- 好感度不足による本編進行停止

探索選択肢を追加する会話は、戦闘を無条件に有利にするものではなく、情報、別ルート、資源、
安全性のトレードオフを増やす。

## 会話選択

- 通常は2〜3択
- 正解・不正解を隠した好感度試験にしない
- 選択前に「情報」「施設提案」「遠征方針」など結果の種類を示す
- 重要な加入と本編会話では、選択による永久拒否を作らない
- 方針選択は後の台詞や探索候補を変えてよいが、兵種や地域を永久に失わせない
- 同じ人物へ繰り返し話しかけて全選択肢を回収する方式は採用しない

初期版では数値の好感度、贈り物、恋愛、結婚を実装しない。人物関係は既読会話と少数の方針フラグで
管理する。

## UI

### 集会所一覧

画面左に話せる相手、右に選択中人物の概要を表示する。

- 未読の重要会話: 黄色の印
- 未読の任意会話: 白い印
- 既読のみ: 印なし
- 加入候補: 「加入会話」表示
- 一時不在: 一覧へ表示せず、理由が重要なら別の人物が説明

一覧の分類タブ:

```text
重要
仲間
開拓民
来訪者
記録
```

人物へカーソルを合わせると、名前、立場、現在話せる内容の短い要約を表示する。未加入人物は
加入候補になるまで名前やシルエットを表示しない。

### 会話画面

- 話者名
- 立ち絵または現行プレースホルダー
- 本文
- 必要時だけ選択肢
- 会話で公開された地域情報・施設情報
- 戻る

本文を自動送りせず、クリックまたは決定入力で進める。履歴ボタンから現在の会話を読み返せる。
会話終了後は集会所一覧へ戻る。

### 通知

拠点帰還時に新規会話数だけ通知する。

```text
集会所に新しい話があります: 3件
```

同じメッセージをモーダル本文とボタンで重複表示しない。重要会話がある場合も集会所を強制的に
開かず、遠征開始時に必要な加入・本編会話だけ確認を出す。

## 最初に実装する会話

灰枝の森の縦切りでは次の8件だけを実装する。

1. 初回遠征前: 行軍隊長と古参守備兵
2. 初回遠征前: 監視弓兵と辺境斥候
3. 林縁から帰還: 負傷した調査員
4. 薬草の沢から帰還: 暁の衛生兵と薬草担当者
5. 灰角大猪攻略後: 遠征隊4人の地域総括
6. 灰枝の森攻略後: 重装兵の加入会話
7. 森の踏査記録登録後: 辺境猟兵の加入会話
8. 開拓拠点発展後: 大工による施設4候補の説明

正式表示名、人物像、関係性は[`cast_reference.md`](cast_reference.md)を正本とする。最初の
システム実装では安定ID、発生条件、既読管理、加入効果、初期8会話の日本語本文を対象にする。

### 初期8会話の確定データ

|---|---|---|---|---|

| `campfire_watch_and_trail` | 新規開始 | ネリス、セラ | 遠方観測と現地踏査の違い | 斥候専用選択肢の存在を予告 |
| `verge_surveyor_returns` | 林縁から安全帰還 | 負傷調査員、トーヴァン | 狼の群れと踏査地点 | 林縁の敵傾向を地域情報へ追加 |
| `hollow_medicinal_report` | 薬草の沢から安全帰還 | トーヴァン、薬草担当者 | 薬草採取と沢の危険 | 薬草群生地と救護テント条件を公開 |
| `forest_company_review` | 灰角大猪撃破、安全帰還 | 初期遠征参加者4人 | 道を確保した意味と残った危険 | 地域総括を記録帳へ追加 |
| `heavy_recruitment` | 重装兵加入候補、宿舎枠あり | ハドリク、マエラ | 労働と戦闘を同じ専門として認める | ハドリク加入 |
| `ranger_recruitment` | 森の踏査記録、宿舎枠あり | ヴェイラ、セラ、ライダン | 街道と生息域を両方記録する | ヴェイラ加入、開拓方針フラグ |
| `first_facility_briefing` | 開拓拠点へ発展 | 大工、オレン加入前は一般工人 | 初期4施設の用途と必要素材 | 4施設候補の詳細を公開 |

### 初期会話の選択

`campfire_command_and_return`:

- 「成果を持ち帰る」: 帰還重視の台詞フラグ
- 「道を先へ伸ばす」: 続行重視の台詞フラグ
- どちらでも機能差はなく、後のライダンとマエラの台詞だけが変わる

`forest_company_review`:

- 「街道を優先する」: 開拓優先フラグ
- 「森への影響を記録する」: 保全重視フラグ
- 「両方を次の遠征条件にする」: 両立方針フラグ
- どの選択でも沈黙した監視所群を解放する

`ranger_recruitment`:

- 地域総括の方針フラグを参照してヴェイラの台詞を変更する
- 加入を一度保留できるが、集会所からいつでも再開可能
- 拒否による永久離脱はない

`first_facility_briefing`:

- 救護、訓練、工作、鍛冶から説明を読む順番だけ選ぶ
- 会話中に建設・素材消費は行わない
- 会話終了後、施設画面へ移動するボタンを表示する

## セーブ

恒久保存:

- 既読会話ID
- 会話選択フラグ
- 公開済み地域情報ID
- 記録帳ID
- 加入処理済み会話ID

保存しないもの:

- 会話中の文字送り位置
- カーソル位置
- 一覧のスクロール位置
- 表示用に導出できる「現在話せる相手」一覧

会話終了時、加入確定時、方針フラグ変更時にオートセーブする。将来Schemaでは未知の会話IDを
保持しても読込を失敗させず、削除済みIDは表示対象から外す。

## 実装順

1. 会話Definition、Condition、Effect、既読状態
2. 条件評価と利用可能会話一覧
3. 集会所一覧UI
4. 会話送り、選択肢、履歴
5. 既読・フラグのセーブ
6. 加入会話をrosterへ接続
7. 地域情報と施設情報の公開
8. 灰枝の森の初期8会話
9. 立ち絵と演出

## 不変条件

- 集会所を任意建設施設にしない
- 任意会話を読まないことで主進行を永久に失わせない
- 重要会話の有無を人物へ順番に話しかけないと発見できないUIにしない
- 会話本文を条件判定に使わない
- Frontier提案の人物をWorld Bibleの確定人物として記述しない
- March Line外側の正体を会話だけで確定しない
