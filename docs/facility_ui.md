# 施設UI仕様

状態: JOJIFrontierの正式な施設画面仕様。施設の状態モデルは
[`base_development.md`](base_development.md)、研究ノードは
[`facility_research.md`](facility_research.md)、製作・所持上限は
[`item_system.md`](item_system.md)を正本とする。

## 画面構成

```text
拠点
-> 施設一覧
   -> 施設詳細
      -> 建設確認
      -> 研究詳細・確認
      -> 製作
```

- 施設一覧は「何があるか」を確認する画面
- 施設詳細は役割、研究、製作への入口
- 建設・研究・製作はそれぞれ別の確定操作にする
- 装備変更はユニット画面、会話は集会所、遠征先選択は司令所から遠征画面へ遷移する
- 稼働切替、停止、解体、再建、施設LvのUIは作らない

## 1. 施設一覧

### 表示順

1. 基礎設備: 司令所、倉庫、宿舎、集会所
2. 建設施設: 診療所、訓練所、工房、鍛冶場

表示順はSaveや建設順で変えない。

### 施設行

各施設を1行で表示する。

| 表示 | 内容 |
|---|---|
| 施設名 | 日本語の正式表示名 |
| 種別 | 基礎設備／建設施設 |
| 状態 | 常設／未建設／建設済み |
| 研究進行 | `研究済み 2 / 5`。研究ノードがない施設では非表示 |
| 次の候補 | 研究可能、条件不足、全研究済みのいずれか |
| 操作 | `訪れる`。未建設施設は`詳細を見る` |

- 一覧へ素材費と長い効果説明を詰め込まない
- 未建設でも施設を非表示にせず、名称・役割・建設条件を確認可能にする
- 未到達地域由来の研究は件数へ含めるが、名称を`未確認の研究`とする
- 新しく建設可能・研究可能になった施設には`新規`表示を付け、詳細を開くと解除する
- 状態は色だけでなく文言とアイコンで区別する

### ホバー

施設名または状態へカーソルを合わせると次を表示する。

- 施設の役割
- 建設済みなら利用可能な主要機能
- 未建設なら建設費と不足条件
- 研究済み数と現在研究可能な数

枠は全文が収まる高さまで拡張し、`...`で説明を切らない。画面外へ出る場合は表示位置を反転する。

## 2. 施設詳細

### 上部

- 施設名
- 基礎設備／未建設／建設済み
- 施設の担当範囲を1〜2行で表示
- `施設一覧へ戻る`

未建設の場合:

- 建設後に使える機能
- 建設費
- 必要開拓段階
- 必要Discovery
- 現在値／必要値
- `建設する`

建設済みの場合:

- `研究`タブ
- 製作担当施設のみ`製作`タブ
- 司令所は`地域情報`
- 宿舎は`ユニットへ`
- 集会所は`話を聞く`
- 倉庫は`倉庫を開く`

### 研究一覧

研究ノードは前提順に縦方向へ並べる。線を多用した巨大ツリーではなく、系統ごとの見出しと行で表示する。

各研究行:

| 表示 | 内容 |
|---|---|
| 日本語名 | 内部IDは通常非表示 |
| 種別 | 製作解放／恒久機能 |
| 状態 | 未発見／条件不足／研究可能／研究済み |
| 効果 | 省略しない短文 |
| 条件概要 | Discovery、素材、前提研究、地域 |
| 操作 | `詳細`。研究可能なら`研究する`も表示 |

- 研究済みノードにも効果確認用の`詳細`を残す
- 未発見研究は効果と固有名を伏せ、取得地域も`未確認地域`とする
- Discoveryを取得済みなら正式名、効果、取得地点を公開する
- 前提研究を選ぶと同じ一覧内の該当行へ移動する
- 共同研究は専用見出しへまとめ、必要な2施設を表示する

## 3. 建設確認

建設確認は中央モーダルで表示し、背後の施設詳細を操作不可にする。

表示内容:

- 建設する施設名
- 建設後に利用可能になる機能
- 消費素材ごとの`所持 / 必要 / 建設後残量`
- 必要開拓段階とDiscovery
- 拠点発展用予約素材を下回る場合の警告
- `建設する`、`戻る`

### 実行規則

1. モーダルを開いた時点では消費しない
2. 確定時に全条件、全素材、未建設状態を再検証する
3. 全条件成立後、素材消費と建設済み登録を1トランザクションで行う
4. 失敗時は素材と施設状態を一切変更しない
5. 成功後は同じ施設詳細へ戻り、解放機能を結果メッセージで表示する
6. 二重クリック、連打、連続保存で二重建設・二重消費しない

予約素材警告は実行禁止ではない。ただしキー素材は指定用途以外の建設に表示も消費もしない。

## 4. 研究詳細・確認

### 詳細表示

- 研究名
- 製作解放／恒久機能
- 完全な効果
- 必要Discoveryと取得地域・地点
- 必要素材と現在数
- 前提研究
- 対応施設
- 研究後に追加されるレシピまたはUI機能

条件表示:

| 状態 | 表示 |
|---|---|
| 充足 | `達成済み` |
| 不足 | `不足`と不足量・不足条件 |
| 未発見 | `未確認の記録が必要` |
| 研究済み | `研究済み` |

### 研究確認

- 素材を消費する研究だけ確認モーダルを出す
- 素材なしの宿舎拡張などは効果確認後、1回の確定で解放する
- 確定時に条件を再検証し、素材消費と研究済み登録を原子的に処理する
- 研究済みノードを再実行できない
- 研究成功後、追加レシピ一覧または恒久機能を結果表示する
- 研究で施設Lv表示や能力値上昇演出を出さない

## 5. 製作画面

### 施設ごとのタブ

| 施設 | タブ |
|---|---|
| 診療所 | 回復／状態治療 |
| 工房 | 戦闘道具／探索道具／帰還支援 |
| 鍛冶場 | 兵種別武器／調整特性 |
| 訓練所 | 製作なし。スキル研究だけ |

鍛冶場は`兵種を選択 -> 武器系統 -> レシピ詳細`の順に掘り下げる。装備者の変更は行わない。

### レシピ行

- 完成品名
- 所有数、装備中数、未装備数
- 効果
- 必要素材
- 必要研究・Discovery
- 製作可能数
- `製作する`

未解放レシピ:

- Discovery取得前は名称と効果を伏せる
- Discovery取得後は不足研究と素材を表示する
- 地域進行に必須のキー素材を製作費として表示しない

### 数量指定

- 消耗品、武器、調整特性は`- / 数量 / +`と`最大`を表示する
- 初期数量は1
- 最大値は素材、倉庫上限99、キー素材保護を反映した最小値
- 探索道具は数量指定を出さず1回だけ製作する
- 0個、上限超過、素材不足では確定できない

### 製作確認

- 完成品と数量
- 合計素材
- 製作後残量
- 倉庫の`現在 / 上限`
- 武器・特性なら「同時装備には人数分必要」を表示
- 探索道具なら「恒久品・重複製作不可」を表示

確定時に全素材と倉庫空きを再検証し、全個数を一括成功または全失敗にする。

## 戻る操作

- 製作確認 -> 製作レシピ
- 製作画面 -> 施設詳細
- 研究確認 -> 研究詳細
- 研究詳細 -> 施設詳細の同じスクロール位置
- 施設詳細 -> 施設一覧
- 施設一覧 -> 拠点

戻る操作で選択・製作・研究・素材消費を発生させない。画面を閉じても最後に見ていた施設とタブは、
拠点滞在中だけ保持する。ゲーム再起動後のUI位置は保存しない。

## 表示・入力規則

- 日本語と英語で同じ情報量を表示する
- 内部ID、Seed、未翻訳キーを通常画面へ出さない
- `??`、欠落Glyph、UTF-8途中切断を自動テストで検出する
- 主要本文は画面Scale適用後18px相当以上、ボタンは20px相当以上を基準とする
- 長文は折り返し、必要なら詳細領域だけスクロールする
- 画面全体を1枚の巨大カードにせず、一覧行と詳細領域を明確に分ける
- 無効ボタンにはホバーまたは選択時に理由を表示する
- キーボード、ゲームパッド、マウスで同じ順序にフォーカス移動できるようにする

## セーブと失敗処理

- 建設、研究、製作成功後にオートセーブ要求を発行する
- 保存成功前でもゲーム内状態は確定済みとして扱い、同操作を再実行させない
- 保存失敗時はHUDへ通知し、再試行可能にする。素材を巻き戻して二重操作可能にしない
- 建設・研究・製作の処理中に画面遷移できない
- ImportまたはSave復元後は、保存された建設・研究・所有状態からUIを再構築する

## 受入条件

1. 未建設／建設済み、未研究／研究済み、未所有／所有済みを混同しない。
2. 稼働、停止、解体、再建、施設Lvを表示しない。
3. 建設・研究・製作の失敗時に一部素材を消費しない。
4. 同じ施設・研究を二重確定できない。
5. 消耗品99、武器99、特性99、探索道具1の上限を超えない。
6. 武器の所有数、装備中数、未装備数が一致する。
7. 未発見研究の名称・地域・効果を先行表示しない。
8. ホバー説明、研究効果、必要素材が枠外で見切れない。
9. 施設一覧から各詳細、研究、製作へ移動し、戻る操作で適切な画面へ戻れる。
10. 建設・研究・製作成功後のSave再読込で状態と在庫が一致する。
