# JOJIFrontier 戦闘不能・敗北・撤退仕様

遠征資産の分類、安全帰還トランザクション、持込品返却、報酬台帳は
[`expedition_rewards.md`](expedition_rewards.md)を正本とする。

## 状態

共通仕様確定、任務目標と手動撤退は未実装。現在実装済みの「味方全滅で敗北」「敗北時に
Pending資産を失う」「帰還時に全回復」を、この文書の共通ルールへ拡張する。

## 戦闘不能

- HPが0になったキャラクターは死亡せず、遠征中の戦闘不能になる
- 戦闘不能になった時点で盤面から取り除き、そのマスを空きマスとして扱う
- 敵味方とも、そのマスを通過・停止できる
- 戦闘不能者は攻撃、スキル、アイテム、地形効果、範囲効果の対象にならない
- 戦闘不能直前に付いていた状態異常、一時強化、クールダウンは消去する
- 戦闘不能状態とHP0は、キャンプおよび次の戦闘へ引き継ぐ
- 次の戦闘では戦闘不能者を出撃させず、残りの生存者だけを初期配置する
- 生存者が1人でもいれば、全滅以外の敗北条件に触れない限り戦闘を続行できる
- キャラクターの恒久死亡、負傷後遺症、装備喪失は採用しない

戦闘不能者の円、名前、HPバーを盤面へ残さない。戦闘ログとパーティHUDには
「戦闘不能」として残し、キャンプで誰を復帰できるか確認可能にする。

## 復帰

| 方法 | 使用場所 | 効果 |
|---|---|---|
| 救命包 | キャンプ | 戦闘不能者1人を最大HPの25%、最低1で復帰 |
| 高級救命包 | キャンプ、将来解放 | 戦闘不能者1人を最大HPの50%、最低1で復帰 |
| 拠点帰還 | 拠点 | 全員の戦闘不能を解除し、HPを最大まで回復 |

- 戦闘中の蘇生は採用しない
- 通常回復、薬草地点、野営食、衛生兵のHealでは戦闘不能を解除できない
- 復帰時に状態異常や一時効果は付かない
- 救命包使用後も同じキャンプで追加回復アイテムを使用できる
- 救命包を使わず、少人数のまま遠征を続行できる

## 勝利条件

任務は次の勝利条件を1つ以上持つ。複数指定時は任務データでANDまたはORを明示する。

| 種類 | 条件 |
|---|---|
| 敵全滅 | 生存している敵ユニットが0 |
| 指揮官撃破 | 指定された敵Unit IDが戦闘不能または撤退 |
| 地点確保 | 指定マスで味方が行動終了し、必要Phase数を維持 |
| 防衛 | 指定ラウンド終了まで防衛対象が健在 |
| 脱出 | 必要人数が指定された脱出マスで行動終了 |
| 装置操作 | 指定ユニットまたは指定兵種が装置マスで行動終了 |

- 初期地域は敵全滅を主目的とし、副目標で地点確保と装置操作を教える
- 勝利条件達成後に不要な敵全滅を要求しない
- 勝利判定は行動、状態異常ダメージ、地形効果、敵撤退の解決後に行う
- 勝利と敗北が同じ処理で成立した場合、護衛対象喪失などの敗北を優先する

## 敗北条件

| 種類 | 条件 | 初期使用 |
|---|---|---|
| 部隊全滅 | 出撃したプレイヤーユニットが全員戦闘不能 | すべての戦闘 |
| 護衛失敗 | 必須護衛対象がHP0になり撤退 | 護衛任務のみ |
| 防衛地点喪失 | 敵が指定マスで行動終了し占領を確定 | 防衛任務のみ |
| ターン超過 | 制限ラウンド終了時に勝利条件未達成 | 明示された任務のみ |
| 重要装置破壊 | 破壊禁止の装置耐久が0 | 対応任務のみ |

- 行軍隊長など特定のプレイアブル兵種が倒れただけでは敗北しない
- 主人公死亡によるゲームオーバーは採用しない
- 護衛対象は死亡せず撤退扱いとし、加入イベントは次回遠征で再挑戦できる
- 敗北条件、ターン制限、対象マスは戦闘開始前と戦闘HUDに常時表示する
- 非公開の即時敗北条件を置かない

## 手動撤退

戦闘中のメニューに「撤退」を置く。誤操作防止の確認画面を必須とする。

- Player Phase中、敵の行動解決中・攻撃演出中・結果画面以外で選択可能
- ユニットを選択中の場合は選択と仮移動を取り消してから確認画面を開く
- 確認画面に失うPending Loot、Pending Discoveries、加入候補を一覧表示する
- 確定すると現在の戦闘を敗北扱いで終了し、拠点へ戻る
- Pending Loot、Pending Discoveries、未確定の地域進行、加入候補をすべて失う
- 使用済みアイテムは戻らない
- 未使用の持ち込みアイテムは恒久在庫へ戻す
- キャラクターは全員、拠点帰還処理で完全回復する
- 撤退した地点は攻略済みにせず、次の遠征で再生成して再挑戦する

キャンプの「拠点へ帰還」は安全帰還であり、手動撤退とは異なる。キャンプ帰還では
Pending資産を正式取得する。帰還信号弾は戦闘外の地点間画面、探索選択前、キャンプで使用でき、
使用時点までのPending資産を確定して帰還する。通常帰還できるキャンプでは通常帰還を優先する。
戦闘中に帰還信号弾を使用して安全帰還することはできない。

キャンプの「拠点へ帰還」(`GameApp::returnToBase()`)は[`campaign_balance.md`](campaign_balance.md)
「初見で途中帰還することを失敗扱いにせず、情報と安全路を持ち帰る正規の進行にする」が指す、
その具体的な実装である。「アイテムを使いすぎて次の戦闘に自信が持てない」と感じたらここで
一旦引く、というのがこのゲームの標準的な攻略手順であり、単発戦闘の難度を下げて解決すべき
問題ではない(詳細は`campaign_balance.md`「灰枝の森: 撤退判断の設計意図」参照)。この判断を
情報面で支えるため、Camp画面には残りBagアイテムの内訳と、Scout Network解放済みなら次地点の
敵構成プレビューを表示する(`GameApp::nextSiteEnemyRosterNames()`)。設定画面の「遠征リタイア」
(`retireExpedition()`)はこの「拠点へ帰還」とは別物で、Pending資産を確定せず全て失う点で
上記の「手動撤退」に近い、より重い選択肢である。

## 敗北時の遠征処理

```text
敗北条件成立
-> 戦闘入力を停止
-> Pending Lootを破棄
-> Pending Discoveriesを破棄
-> 未確定Region Progressを破棄
-> 加入候補を破棄
-> 敗北結果を保存
-> 敗北画面
-> 拠点へ戻る
-> 全員のHPと戦闘不能を完全回復
```

- 恒久Storage、Discovery Registry、建設済み施設、加入済み人物、製作済み装備は失わない
- 遠征中に消費したアイテムは戻さない
- 未使用の持ち込みアイテムはStorageへ戻す
- 敗北地点までの戦闘勝利数や踏査数は恒久進行へ加算しない
- 統計には到達地点、戦闘勝利数、戦闘不能者数、失ったPending資産を記録できる

## ボス戦と再挑戦

- ボス戦から撤退または敗北した場合、ボスHP、状態、破壊した障害物を次回へ持ち越さない
- 次回遠征では地域シードを新しく生成し、地形と通常配置を作り直す
- ボスの基本能力、行動規則、探索ルートで固定されたギミックは変えない
- ボス撃破フラグと報酬は勝利時にPendingへ入り、安全帰還まで恒久化しない
- ボス撃破後に遠征を続けて全滅した場合も、未確定のボス報酬と攻略進行を失う
- 地域を一度安全帰還で攻略済みにした後は、再戦敗北で攻略済みフラグを失わない

## 任務対象と副目標

- 主目的失敗は戦闘敗北になる
- 副目標失敗だけでは戦闘を終了しない
- 踏査地点、追加資源、加入候補、無傷防衛などは原則として副目標にする
- 副目標報酬は条件達成後も戦闘勝利までPendingへ追加しない
- 戦闘勝利後に達成済み副目標を一括評価し、Pendingへ追加する
- 加入候補が撤退しても敵全滅で通常報酬を得られるが、その人物は加入しない

## セーブと敗北回避対策

戦闘途中の任意セーブは対象外とする。ただしブラウザ終了で敗北を無効化できないよう、
戦闘開始直前に遠征ジャーナルを保存する。

```cpp
struct ActiveBattleMarker {
    std::string expeditionAttemptId;
    std::string regionId;
    std::string nodeId;
    std::uint64_t battleSeed;
    bool resolved = false;
};
```

1. 探索結果と戦闘Seedを確定する
2. `ActiveBattleMarker{resolved=false}`を同期保存する
3. Web版は同期完了を確認してから戦闘を開始する
4. 勝利または敗北の結果処理と同時に`resolved=true`を保存する
5. 未解決Markerを読み込んだ場合は「中断された戦闘」として扱う

初期方針では、中断された戦闘は同じSeed・同じHP・同じ消耗品・同じ配置で戦闘開始時から
再開する。戦闘途中の盤面は復元しない。再挑戦しても乱数結果を変えるためのSeed再抽選は行わない。

- 戦闘開始前のHPやアイテムへ巻き戻せないよう、Markerと同じ保存トランザクションへ含める
- 勝敗結果を保存する前に敗北画面からアプリを終了しても、次回起動時に結果処理を再実行する
- 結果処理には`expeditionAttemptId`を使い、報酬付与や資産破棄を二重実行しない
- 保存失敗時は戦闘を開始せず、再試行または拠点帰還を表示する
- 開発用の戦闘再開・Seed変更は製品版UIへ出さない

## UI

- パーティHUDでは戦闘不能者を暗色表示し「戦闘不能」と表示する
- 戦闘開始画面に勝利条件、敗北条件、副目標、ターン制限を分けて表示する
- 戦闘HUD端に主目的と残りラウンドを常時表示する
- 撤退確認では損失を赤、拠点で回復する内容を通常色で表示する
- 敗北画面には敗北理由、失った資産、拠点で全回復することを表示する
- 「撤退」「安全帰還」「帰還信号弾」を同じ文言や同じ色で表示しない

## 実装時の不変条件

- キャラクターを恒久死亡させない
- 戦闘不能者を障害物として利用できない
- 主目的以外の副目標失敗で即敗北させない
- プレイヤーへ事前表示していないターン制限を発生させない
- 手動撤退でPending資産を確定しない
- 安全帰還でPending資産を失わない
- 敗北で恒久Storageや登録済みDiscoveryを巻き戻さない
- 敗北直前の終了や再読み込みでSeedと戦闘開始状態を再抽選しない
