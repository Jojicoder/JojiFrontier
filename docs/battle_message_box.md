# 戦闘メッセージボックス仕様

状態: 実装用の共通UI・通知仕様として確定。Objective判定と報酬付与は行わず、確定済みの
BattleEvent、Objective状態、BattleOutcomeを日本語で表示する。

行動確定前の数値、攻撃範囲、反応、危険予告は[`combat_forecast.md`](combat_forecast.md)を正本とし、
本書の履歴へ未確定の予測を追加しない。

地点別の初回説明は[`stage_facility_progression.md`](stage_facility_progression.md)の
「戦闘開始メッセージ要旨」、詳細条件は`regions/`以下を参照する。

## 表示する情報

- 戦闘開始: 地点名、主目的、敗北条件、公開済み副目標、重要な地形・Objectを1回表示
- 戦闘中: 目標進捗、目標失敗、増援、Object耐久警告、ボス攻撃予告、地形変化
- 状態: 毒・炎上・移動低下・防御低下・よろめきの付与と解除、薬草回復、使用不可アイテム
- 戦闘終了: 勝利・敗北理由、達成した副目標、Pendingへ入った報酬と重要発見

通常攻撃ごとのダメージ、カーソル操作、全移動、敵AIの思考過程は表示しない。既存の短い
ダメージ・回復バナーは瞬間フィードバックとして残せるが、このメッセージ履歴には入れない。

## レイアウト

- 戦闘HUD下端に固定し、盤面と行動ボタンへ重ねない
- 通常は2行、詳細展開時は最大5行、履歴は新しい順に20件
- 長文は領域幅で折り返し、省略記号でObjectiveや敗北条件を切らない
- メッセージの増減で盤面、行動ボタン、ユニット情報の位置を動かさない
- 重要度は情報、目標、警告、重大、結果の5段階。色だけでなく見出し・アイコンも変える

## 入力を止めるメッセージ

確認ボタンを必要とするのは次の4種だけとする。

1. 戦闘開始ブリーフィング
2. 新しい主目的または敗北条件の追加
3. ボス固有行動を初めて説明するとき
4. 勝利・敗北結果

副目標、状態異常、通常増援、回復、Object耐久警告では入力を止めない。

## 優先順位

同じイベントBatchでは次の順に並べる。

1. 敗北
2. 勝利
3. 必須Objective失敗
4. ボス予告
5. 必須増援・地形変化
6. 主目的進捗
7. 副目標
8. Object耐久
9. 状態・回復・アイテム

敗北と勝利が同じBatchに成立した場合は既存ルールどおり敗北を表示し、勝利通知を出さない。

## 地域固有メッセージ

- 灰枝の森: 狼の集中対象、茂みの初回回避説明、薬草撤収条件、大猪の突進列、倒木衝突、
  激昂、踏査地点、薬草地点
- 沈黙した監視所群: 記録箱、信号装置、主信号機、信号封鎖
- 灰鉄採石場: 支柱耐久、巻上機、崩落予定マス、潜行位置
- 黒水低湿地: 毒地形、水門操作、浅瀬への変化、潜行位置
- 風裂き高原: 強風方向、押し出し先、護衛の分断、敵退路
- 旧辺境集落: 中立住民、交渉継続条件、退避路、増援波
- 燼火峡谷: 戦場熱量、炎上床、冷却水路、過熱装置
- 埋没聖堂: 避難者、救護台、写本箱、封鎖輪、降伏条件
- 破砕された前線砦: 城門・予備壁、増援方向、命令箱、隊長撤退条件
- 地図外縁: 未知の反応は正体を断定せず表示、標識数、帰還基点、脱出条件

## 重複防止

- `BattleEventId`を消費済み集合で管理し、同じイベントを2回表示しない
- Objective達成・失敗はObjective IDごとに1回だけ表示する
- Object耐久は初回、50%以下、25%以下、0の閾値を跨いだ時だけ表示する
- 地形警告は実際に状態が変わった時だけ表示する
- 同一Batchの同名状態付与は対象名をまとめ、1件に集約できる
- BattleOutcome確定後は結果とPending以外の新規メッセージを受け付けない

## ローカライズとデータ契約

```cpp
enum class BattleMessagePriority { Info, Objective, Warning, Critical, Result };

struct BattleMessage {
    std::uint64_t messageId;
    std::optional<std::uint64_t> sourceEventId;
    std::string textKey;
    std::vector<std::string> arguments;
    BattleMessagePriority priority;
    bool requiresConfirmation = false;
    float visibleSeconds = 3.0F;
};
```

C++へ日本語本文を直書きせず、`textKey`と引数をローカライズ層へ渡す。英語でもイベントID、
数値、判定結果は同一にする。通知層はHP、Objective、報酬を変更しない。

通常の一時メッセージ履歴はセーブしない。確認待ち中に中断保存を許可する将来仕様では、確認待ち
メッセージの`textKey`、引数、発生元イベントIDだけを保存する。

## 受入テスト

1. 同じBattleEventを2回渡しても表示は1回。
2. 同一Batchで勝利と敗北が成立したら敗北だけ表示。
3. 日本語・英語で折返し後も盤面や行動ボタンと重ならない。
4. 突進、増援、地形変化は実行前に警告される。
5. ローカライズを変えてもObjective判定と報酬は変化しない。
6. 確認必須4種以外は戦闘入力を止めない。
7. Pending報酬を再描画しても二重付与・二重表示しない。
8. 地図外縁の未知対象は、正体判明イベント前に固有名を表示しない。
