# 敵AI共通評価仕様

文書種別: **正本**
正本範囲: AI候補、Score、役割、小隊予約、撤退、難易度差、決定論

## 状態

共通仕様確定。敵AIの候補生成、任務評価、役割分担、撤退、情報制限、決定論的同点処理を定める。
地域書と敵DefinitionはProfile、役割、任務、固有行動だけを指定する。

## 目的

- 敵が全員同じ最短距離攻撃を選ぶ状態をなくす
- 野生動物、人間勢力、魔物を同じ判断にしない
- 敵全滅以外の任務で、敵もObjective達成へ行動する
- 高難度を能力値補正や非公開情報の使用ではなく、評価精度と連携で作る
- 同じBattleStateとSeedから同じ行動を再現する

## 攻撃性の原則

- 攻撃可能なら攻撃する
- 攻撃できない場合は、攻撃可能になる位置へ移動する
- 移動後に攻撃、スキル、予告行動を再判定する
- 移動だけで終了するのは、移動しても攻撃手段を実行できない場合だけ
- 何もせず待機できるのは、スタン、完全な移動不能、対象消失など行動不能時だけ
- ボスの攻撃予告は待機ではなくスキル行動として扱う

## データモデル

```cpp
enum class AiFactionProfile {
    Wildlife,
    Bandit,
    Mercenary,
    RivalExpedition,
    RegularArmy,
    Monster,
};

enum class AiRole {
    Defender,
    Striker,
    Ranged,
    Support,
    Flanker,
    Engineer,
    Courier,
    Boss,
};

struct AiDefinition {
    AiFactionProfile faction;
    AiRole role;
    std::string missionProfileId;
    int aggression;
    int selfPreservation;
    int cohesion;
    int pursuitLimit;
    bool mayRetreat;
    bool maySurrender;
};
```

値は0〜100。能力値や兵種データへAI固有の表示文を埋め込まない。

## 行動生成

各Unitについて、現在のルールで合法な次の候補を生成する。

- 移動せず攻撃、Skill、Object操作、待機
- 到達可能な各停止マスから攻撃、Skill、Object操作、待機
- 任務上許可された撤退・降伏
- 予告固定Boss行動

通過不能、停止不能、射程外、使用回数0、未発見対象を使う候補は生成しない。候補生成時点で
味方Unitのいるマスは通過可能・停止不可、敵Unitのいるマスは通過・停止不可を適用する。

候補数が多い場合も距離だけで早期切捨てせず、次を最低1件ずつ残す。

- 最大任務進行
- 最大ダメージ
- 最も安全な停止位置
- 役割に合う位置
- 撤退・支援

## 強制行動

次は通常Score比較より先に処理する。

1. 予告済みBoss行動の実行
2. Scriptで固定された離脱・増援・Phase移行
3. 敗北を避けるため必須のObject防衛・脱出妨害
4. 通常候補評価

強制行動が実行不能な場合は固有仕様の失敗処理を使い、その場で最も近い対象へ無料攻撃しない。

## 共通Score

全候補は整数Scoreで比較する。

```text
score = mission
      + outcome
      + damage
      + control
      + position
      + role
      + cohesion
      - danger
      - overkill
      - missionAbandonment
```

基準値:

| 評価 | Score |
|---|---:|
| この行動で敵側主目的を達成 | +10000 |
| 次のPlayer Actionで確定する敵側敗北を防ぐ | +8000 |
| 必須Objectiveを1段階進める | +3000 |
| 必須Objectへの即時脅威を除く | +2200 |
| 対象Unitを戦闘不能にする | +1500 |
| 与ダメージ1 | +40 |
| 有効な状態異常を付与 | +250 |
| 味方Unitを戦闘不能から救う・撃破圏外へ戻す | +600 |
| 役割に適した位置へ移動 | 最大+400 |
| 次Player Phaseに撃破される危険 | 最大-1200 |
| 防衛持ち場を追撃上限より離れる | -1000 |
| 既に予約ダメージで倒れる対象への過剰攻撃 | 最大-900 |

地域Mission ProfileとFaction Profileは項目の重みを変更できるが、即時勝敗の優先順位を逆転させない。

## 対象評価

同条件では次を基本とする。

1. この行動で撃破可能
2. 敵側Objectiveを妨害している
3. 孤立している
4. 低HP
5. 支援・指揮役
6. 少ない移動コストで攻撃可能
7. 安定Unit ID

Faction差分:

- 野生動物: 孤立と縄張り侵入を高評価。人工装置を操作しない
- 野盗: Loot Container、低HP、離脱経路を高評価
- 傭兵: 契約Objectiveと生存を高評価。無意味な特攻を避ける
- 競合開拓団: 資源地点、経路、装置の先取を高評価
- 正規兵: 指揮官、防衛線、小隊陣形、規律ある後退を高評価
- 魔物: 種別固有行動と地形変化を優先し、人間の士気判断を流用しない

## 役割

| 役割 | 優先 | 行わないこと |
|---|---|---|
| Defender | 狭路、Object、後衛の保護 | 持ち場を越えた長距離追撃 |
| Striker | 撃破可能対象、前線突破 | 後衛より後ろで待機し続ける |
| Ranged | 最小射程維持、安全な射線 | 理由なく隣接位置へ進む |
| Support | 複数支援、負傷者、解除 | 通常攻撃を支援より常に優先 |
| Flanker | 上下行、孤立対象、退路 | 正面の高DEFへ全員集中 |
| Engineer | 装置、障害物、増援口 | 野生動物Profileへ付与しない |
| Courier | 伝令、離脱、Objective起動 | 撃破目的だけに留まる |
| Boss | 予告行動、固有Objective | 通常Scoreで予告を変更 |

同じ役割が多い場合でも、1つの対象や停止マスへ全員を予約しない。

## 小隊予約

Enemy Phaseは`aiPriority`降順、安定Unit ID昇順で1体ずつ解決する。各行動後にBattleStateを更新し、
残りUnitの候補を再計算する。

Phase中は次を予約する。

- 停止予定マス
- 対象への予定ダメージ
- 支援予定対象
- 防衛担当Object
- 上下行の進入担当数

予約ダメージで対象HPが0以下なら追加攻撃へ過剰攻撃Penaltyを付ける。ただしBoss撃破、回復可能な
対象、確実なObjective達成では必要な集中を許可する。予約はSave対象にせず、復元したPhase開始時に
確定BattleStateから再計算する。

## 地形と危険

- プレイヤーの公開済み攻撃範囲を危険として評価
- 茂み、狭路、高所相当地形など役割に合う地形を評価
- 罠は敵が発見済みの場合だけ回避
- 煙幕内の正確な未発見位置を参照しない
- 予告済み地形変化を認識し、即時任務達成より低い範囲で危険Penaltyを付ける
- 逃げ場がない場合も行動放棄せず、最良の合法候補を選ぶ

敵はプレイヤーの未確定入力、非公開バッグ、未設置罠、乱数結果を参照しない。

## 撤退と降伏

共通初期値:

- 人間系はHP25%以下で、即時任務達成がなく、安全なExitへ到達可能なら撤退候補を生成
- 野生動物はHP20%以下、または群れの主を失った時に縄張り外への逃走候補を生成
- 正規兵・傭兵は指揮官喪失時に`cohesion`を30低下
- 退路がなく、`maySurrender = true`で士気条件を満たす人間系は降伏候補を生成
- 魔物、狂信者、召喚物はDefinitionで許可しない限り撤退・降伏しない
- Bossは地域書のScripted Withdrawal以外では通常撤退判定を使わない

撤退・降伏を撃破として扱わない。Objectiveが許可する退場理由だけを達成へ加算する。

## 難易度との関係

初期版では難易度選択を実装しない。将来追加する場合も、AIは次だけを変える。

- 候補保持数
- 1手または2手先の公開情報による評価
- 小隊予約の精度
- 役割とMission Weightの適用精度

敵だけ命中率、HP、STRを隠れて上げたり、未公開情報を読ませたりしない。

将来の難易度Profileは数値を次の範囲に限定する。

| 項目 | 標準 | 高難度で許可する範囲 |
|---|---:|---:|
| 候補保持数/Unit | 12 | 16〜24 |
| 先読み | 現在Action | 公開情報による次の1 Actionまで |
| 集中攻撃予約 | Damageのみ | Damage、状態、回復を含む |
| Objective Weight | 100% | 110〜125% |
| Cohesion Weight | 100% | 110〜125% |
| Danger Penalty | 100% | 110〜125% |

HP、STR、DEF、敵数、増援数はAI難易度では変更しない。地域側の高危険度任務として変更する場合は、
遠征開始前に明示する。

## 決定論

Scoreが同じ場合は次の順で決定する。

1. Mission進行量が大きい
2. 戦闘不能にできる
3. 受ける予測ダメージが小さい
4. 移動コストが小さい
5. 対象Unit ID昇順
6. 停止位置を行、列の順
7. Action Definition ID昇順

同点解決にフレーム時刻やコンテナ走査順を使わない。明示的なランダム行動だけBattle Seedと
AI Decision Indexを使い、結果をSave Snapshotへ固定する。

## UIと説明可能性

- 敵のScoreや全候補は表示しない
- 敵が守るObject、離脱地点、増援起動条件はObjectiveまたは戦場表示で理解可能にする
- 重要行動は「物資箱を狙っている」など短い意図アイコンを表示できる
- Boss予告、強力な増援、即時敗北につながる操作は事前に警告
- 行動後のメッセージは結果だけを表示し、内部思考ログを出さない

## 受入条件

- 同じBattleStateとSeedで同じ行動順と対象を選ぶ
- 狼が孤立対象を優先し、人工装置を操作しない
- Defenderが追撃上限を越えて持ち場を放棄しない
- Rangedが攻撃可能なら最小射程内へ自ら入らない
- 予約ダメージにより全敵がHP1の同じ対象へ過剰集中しない
- 敵が自身の主目的達成を単純ダメージより優先する
- 未発見罠、非公開バッグ、未来入力を参照しない
- 撤退、降伏、撃破をObjectiveと報酬で区別する
- 予告済みBoss行動を通常Scoreで変更しない
