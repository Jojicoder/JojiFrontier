# JOJIFrontier 戦術設計リファレンス

文書種別: **説明文書**
参照正本: [`class_reference.md`](class_reference.md)、[`skill_system.md`](skill_system.md)、
[`enemy_ai_rules.md`](enemy_ai_rules.md)、[`mission_objectives.md`](mission_objectives.md)
この文書は設計意図とレビュー観点を説明し、数値・安定ID・例外を定義しない。

## 目的

この文書は、兵種、武器、スキル、敵AI、地形、マップ、副目標を追加するときの参考書とする。
実装状況は `implementation_status.md`、戦闘数値は `battle_system.md`、兵種の役割境界は
`class_reference.md`、武器分岐は `base_development.md`を参照する。

JOJIFrontierでは次の4種類の勝ち方を成立させる。

1. 敵を直接倒す
2. 敵の移動と選択肢を制限する
3. 敵全滅以外の目的を達成する
4. HPとアイテムを温存して次の戦闘へ進む

## 基本原則

- 敵、地形、探索選択、副目標に合わせて4人を選ばせる
- 防御役、機動役、遠距離役、支援役の内部でも役割を重複させない
- 攻撃位置だけでなく、行動終了後に残る位置を重要にする
- ランダム地形によって毎回異なる進路判断を作る
- 戦闘単体より遠征全体のHP・消耗品・戦闘不能リスクを優先できるようにする
- 敵AIは通路、罠、攻撃範囲、孤立を評価する

## 戦術パターン

### 前線形成

- 古参守備兵が敵の移動継続を止める
- 重装兵が集中攻撃を受ける
- 槍兵が移動してきた敵を迎撃姿勢で受ける
- 工兵が防護板や障害物で通路を作る
- 監視弓兵と衛生兵が後方から支援する

必要条件: 狭路、迂回路、後衛を狙う敵AI。

### 誘導と分断

- 重装兵を危険範囲へ置いて攻撃を誘う
- 古参守備兵が後続を止める
- 制圧弓や猟兵の罠で別の敵を遅らせる
- 孤立した敵へ集中攻撃する

必要条件: 敵AIが孤立、低HP、到達可能な味方、危険地形を評価する。

### 攻撃範囲による追い込み

- 監視弓兵の射程で安全な移動先を減らす
- 猟兵の罠で逃げ道を危険にする
- 工兵の障害物で進路を限定する
- 出口を古参守備兵や槍兵が守る

必要条件: 敵AIが攻撃範囲、罠、障害物を認識する。

### ノックバック連携

- 重槍で敵を茂みや監視所から外す
- 味方の集中攻撃範囲へ押し込む
- 副目標や味方への接近を遅らせる
- ボスを倒木などの専用障害物へ誘導する

必要条件: 押し出し先の通行・占有判定、ボス耐性、地形衝突効果。

### 行動終了地点の選択

- 薬草地点で行動終了して回復する
- 茂みで行動終了して回避を得る
- 行軍隊長の支援範囲内へ戻る
- 槍兵を敵の突撃経路へ置く
- 古参守備兵を狭い通路へ置く

必要条件: 地形効果、支援範囲、敵攻撃範囲を確定前に確認できるUI。

### 側面攻撃と離脱

- 辺境斥候が悪路を越えて側面へ進む
- 伝令騎兵が行動後に再移動する
- 行軍隊長が少人数の側面部隊へ追従する
- 監視弓兵が反対側から退路を制限する

境界: 斥候は悪路突破、伝令騎兵は再移動を担当し、両方を同じ兵種へ与えない。

### 射程支配

- 監視弓兵が遠距離直接火力を担当する
- 長槍が後列から単体を攻撃する
- 制圧弓が敵MOVを低下させる
- 戦闘魔導士が密集した敵と低RESの敵を攻める

境界: 弓兵は単体直接火力、猟兵は罠、魔導士は限定範囲攻撃を担当する。

### 隊形選択

- 行軍隊長を中心に少人数で固まる
- 旗手を中心に広く展開する
- 薙ぎ払いを持つ敵には分散する
- 突進予告の列を空け、障害物へ誘導する

必要条件: 敵の範囲攻撃と予告を選択中の盤面へ表示する。

### 持久戦

- 重装兵が攻撃を受ける
- 古参守備兵が接敵数を制限する
- 衛生兵が単体回復と状態異常治療を行う
- 工兵が防護板を置く
- 監視弓兵が安全圏から攻撃する

評価基準: 勝利ターン数だけでなく、終了時HP、戦闘不能者、消耗品使用数を記録する。

### 速攻

- 決闘剣で孤立した敵を倒す
- 戦弓で最大HPの敵へ初撃を与える
- 斥候や伝令騎兵が敵後衛へ接近する
- 戦闘魔導士が密集した敵を同時攻撃する

代償: 隊形が崩れ、失敗時に孤立や集中攻撃を受けやすくする。

### 副目標攻略

- 斥候が踏査地点へ先行する
- 伝令騎兵が救出対象、装置、戦利品地点へ向かう
- 工兵が橋や障害物を処理する
- 残りの味方が敵を引きつける
- 達成後に合流または撤退する

副目標例: 地点で行動終了、一定ターン防衛、対象救出、装置停止、戦利品回収、指定地点離脱。

### 地形利用

- 灰地と浅瀬は移動経路を変える
- 茂みは防御位置になる
- 薬草地点は回復と行動終了を交換する
- 瓦礫と障害物は通路を限定する
- 倒木などの地域ギミックはボス行動へ干渉する

地域ルール: シンダーウォッチ方面には茂みを生成しない。

## 敵編成への対応

| 敵の特徴 | 有効な編成・対処 |
|---|---|
| 高MOV・突撃 | 槍兵、古参守備兵、制圧弓 |
| 高DEF | 戦闘魔導士、位置操作、副目標優先 |
| 遠距離攻撃 | 辺境斥候、伝令騎兵、障害物利用 |
| 密集集団 | 辺境工兵、辺境猟兵、戦闘魔導士 |
| 単体ボス | 重装兵、衛生兵、行軍隊長、専用ギミック |
| 多数の獣 | 辺境猟兵、槍兵、監視弓兵。恒常大ダメージ特効だけで解決しない |

## 敵戦術設計

候補生成、Score、役割、小隊予約、撤退、情報制限、同点処理の実装契約は
[`enemy_ai_rules.md`](enemy_ai_rules.md)を正本とする。本節は戦術意図とレビュー観点を扱う。

### 現在の実装

現行AIは最も近い生存中の味方を選び、射程内なら攻撃し、届かなければ距離が縮まるマスへ
移動してから再度攻撃を試みる。低HP、孤立、地形、罠、役割連携、副目標は未評価。

将来の敵行動は次の4層を合成して決める。

1. 個体: 兵種・生物ごとの行動特性
2. 小隊: 陣形と味方同士の連携
3. 任務: 防衛、奪取、遅延、離脱などの戦闘目的
4. 勢力: 野生動物、野盗、傭兵、正規兵、魔物ごとの価値判断

### 基本対象評価

敵ごとの個性を優先したうえで、同条件の基本評価順は次とする。

1. その行動で撃破可能
2. 孤立している
3. 低HP
4. 衛生兵・旗手など重要支援役
5. 副目標を確保または妨害している
6. 移動・攻撃コストが低い
7. UnitIdによる決定的な同点処理

すべての敵が同じ優先順位を使わない。狼は孤立、野盗は戦利品、正規兵は任務地点を高く
評価するなど、勢力と個体で重みを変更する。

### 防衛

- 指定地点、橋、装置、衛生兵、指揮官を守る
- 前衛は追撃上限を持ち、持ち場から離れすぎたら戻る
- 弓兵と支援役は前衛の後ろを維持する
- 防衛側が全員プレイヤーへ突撃しない

### 待ち伏せ

- 一部の敵は開始時に非公開
- 指定マスへの侵入、副目標調査、一定ターンで出現する
- 足跡、物音、不自然な地形など事前の兆候を表示する
- 斥候ルートや偵察効果で事前発見できる
- セーブ再読込で出現位置や条件を再抽選しない

### 増援

- ターン経過、装置未停止、敵伝令の離脱など明確な条件で出現する
- 伝令兵や装置を止めると増援を阻止できる
- 出現地点と残りターンを可能な範囲で予告する
- 無限増援は専用任務を除いて使用しない
- 初見で防ぎようのない即時出現攻撃を行わない

### 撤退と再編

- 低HPの敵は衛生兵、防衛地点、離脱地点へ下がる
- 弓兵は隣接される前に前衛の後ろへ移動する
- 指揮官撃破や任務失敗で陣形を縮小する
- 退路がない場合だけ最後まで戦う
- 撤退中も危険な反撃が可能なら無意味な移動を避ける

### 任務優先

敵側もプレイヤー全滅以外の目的を持つ。

- 戦利品を奪って離脱する
- 装置を起動する
- 橋、補給地点、防護板を破壊する
- 指揮官、負傷者、荷運びを護衛する
- 一定ターン時間を稼ぐ
- プレイヤーより先に資源地点を確保する

任務行動には画面上の目標表示と失敗条件を用意し、敵だけが知る隠し勝利条件にしない。

### 役割連携

- 守備兵が通路を止める
- 槍兵が守備兵の後ろで迎撃姿勢を取る
- 弓兵が後方から攻撃する
- 衛生兵が負傷者へ近づく
- 旗手が複数の敵を支援できる位置を維持する
- 騎兵が正面を避けて側面へ回る
- 工兵が進路を狭める場所へ設置物を置く

敵を独立した1体としてだけでなく、2〜4体の小隊として評価する。

### 地形利用

- 弓兵は監視所など射撃に適した位置を優先する
- 軽装敵は茂みを利用する
- 重装敵と守備兵は狭路を利用する
- 工兵は障害物で進路を変える
- 魔導士は複数対象を攻撃できる位置を探す
- 野生動物は人工的な監視所や軍用陣地を戦術的に使わない
- 地域に存在しない地形をAI前提にしない

### 陣形

| 陣形 | 構成と目的 |
|---|---|
| 防衛線 | 前衛、槍兵、弓兵の順で並び、後衛を守る |
| 包囲 | 上下の行から接近し、孤立した味方を狙う |
| 楔形 | 重装兵を先頭に中央の通路を突破する |
| 散開 | 範囲攻撃を受ける敵が互いの距離を取る |
| 護衛 | 指揮官、衛生兵、任務対象を中央または後方へ置く |

### 士気と戦闘終了

- 指揮官撃破で一部の敵が撤退または防御的になる
- 短時間に仲間を失うと陣形を縮小する
- 獣は縄張りの主の撃破や重傷で逃走できる
- 狂信者、召喚物、ボス護衛など撤退しない敵は個別指定する
- 降伏・撤退は撃破と報酬・副目標判定を区別する

### 勢力別の戦い方

| 勢力 | 優先する行動 | 避ける行動 |
|---|---|---|
| 野生動物 | 縄張り防衛、群れ、孤立対象、重傷時の逃走 | 軍用陣地、複雑な装置、組織的な増援 |
| 野盗 | 低HP、戦利品、副目標の奪取、逃走 | 犠牲を前提にした長期防衛 |
| 傭兵 | 隊形、契約目標、負傷時の撤退 | 勝敗と無関係な特攻 |
| 競合開拓団 | 資源地点、装置、経路確保 | プレイヤー全滅だけへの固執 |
| 正規兵 | 指揮官、小隊陣形、任務地点、規律ある後退 | 個体ごとの無秩序な追撃 |
| 魔物 | 種別固有の特殊行動、地形・状態変化 | 人間と同じ士気・護衛判断の流用 |

Embermarch系正規兵を敵として出す場合は、封鎖、迎撃姿勢、後衛保護、規律ある後退を中心に
し、単純な高ステータス軍団にはしない。

### 情報と公平性

- 敵は現在見えている情報と、その敵が発見済みの情報だけを使う
- 未発見の罠、煙幕内の正確な位置、未来のプレイヤー入力を参照しない
- 斥候、指揮官、獣など知覚範囲の違いはデータとして定義する
- ボスの大技と強力な増援は事前に予告する
- 同じ盤面状態とseedでは同じ行動を選ぶ
- 難易度はステータス増加より、対象評価と小隊連携の正確さで上げる
- 高難度でも敵に非公開情報を与えない

### 敵AI実装レビュー

- 敵が何を守り、何を達成しようとしているか画面から理解できるか
- プレイヤーの封鎖、罠、攻撃範囲、地形操作を認識するか
- 前衛と後衛が役割に合った距離を保つか
- 全敵が同じ対象へ不自然に殺到しないか
- 追撃上限と撤退条件があるか
- 待ち伏せ、増援、大技に回避または阻止手段があるか
- 同条件で行動が再現可能か
- AIが知らない情報を使っていないか
- ボスを移動阻害で永久停止できないか
- 敵戦術に対して2種類以上のプレイヤー側対処があるか

| 方針 | 4人編成 | 狙い |

| 安定防衛 | 行軍隊長、古参守備兵、監視弓兵、暁の衛生兵 | 封鎖、後衛火力、回復 |
| 突撃対策 | 古参守備兵、槍兵、監視弓兵、暁の衛生兵 | 移動停止と迎撃姿勢 |
| 高機動 | 行軍隊長、辺境斥候、伝令騎兵、監視弓兵 | 側面、副目標、離脱 |

| 長期遠征 | 重装兵、行軍隊長、暁の衛生兵、監視弓兵 | HPと消耗品の温存 |
| 探索重視 | 辺境斥候、辺境工兵、辺境猟兵、暁の衛生兵 | 探索選択肢と副目標 |

## 武器選択による変更

同じ兵種でも基本武器と分岐武器で担当を調整する。ただし`class_reference.md`の役割境界を
越えて別兵種の中心能力を獲得させない。

槍兵の例:

- 鉄の槍: 欠点のない標準型
- 長槍: 後列から攻撃する射程型
- 重槍: ノックバックで隊形を崩す突破型
- 迎撃槍: 移動してきた敵を受ける迎撃型

## 実装レビュー用チェックリスト

- どの戦術パターンを増やす機能か説明できるか
- 既存兵種の中心役割を奪っていないか
- 明確な代償または不向きな状況があるか
- 敵AIがその機能を認識できるか
- 範囲、効果、発動条件を行動確定前に確認できるか
- ランダム地形でも進行不能にならないか
- ボス専用行動を永久封鎖できないか
- 遠征終了時の損耗へ意味があるか
- 敵全滅以外の目的でも価値があるか
- 日本語と英語で同じルールを説明できているか

状態異常を使う戦術は[`status_effects.md`](status_effects.md)の確定ルール、ボス補正、解除手段に従う。
