# JOJIFrontier 62地点ルートグラフ

## 状態

本編10地域、合計62地点の接続、分岐、合流、キャンプ、安全通過、地点省略、最終地点到達条件、
帰還信号弾の使用位置を定める正本。地点内の戦闘、探索3択、報酬、地形は`docs/regions/`以下の
地域専用書を正本とする。C++とSaveへ落とすデータ契約は
[`route_graph_data.md`](route_graph_data.md)を正本とする。

この文書のグラフはJOJIFrontier固有のゲーム進行であり、World Bibleの地理的な共有正史を
追加・変更しない。

## Nodeと遷移

```cpp
enum class RouteNodeKind {
    RegionEntrance,
    Site,
    Camp,
    Junction,
    RegionExit,
};

struct RouteEdge {
    RouteNodeId from;
    RouteNodeId to;
    std::vector<RouteConditionId> conditions;
    bool allowsReturnFlare = false;
};
```

- 遠征は必ず`RegionEntrance`から始まり、Campから恒久再開しない
- `Site`勝利後にMissionFlowがEdge条件を再評価する
- 分岐の攻略順は遠征をまたいで保持し、安全帰還済み成果だけを恒久条件に使う
- 合流条件がANDの場合、片方だけ攻略してCampから帰還できても後半へは進めない
- 未踏地点はグラフから消さず、司令所の情報解放に応じて名称または「不明地点」を表示する
- 安全通過はNode削除ではなく、同じSite Nodeを戦闘なしで解決するEdge Actionとする
- 省略した地点から初回報酬、再採取報酬、Objective成果を得ない

## 記号

- `S1`: 地点
- `C1`: キャンプ
- `J`: 分岐・合流
- `[RF]`: 帰還信号弾を消費して安全帰還できる地点間状態
- `=>`: 条件付き遷移
- `~>`: 経路確保後に安全通過できる地点

Campでは消費なしの通常帰還を使用するため、帰還信号弾ボタンを無効化する。

## 地域別グラフ

### 1. 灰枝の森

```text
入口 -> S1 灰枝の林縁 [RF]
     -> S2 薬草の沢 -> C1
     -> S3 折れ木の縄張り
     -> 地域出口
```

- C1はS2勝利後
- S1安全通過: 踏査地点を確保して安全帰還
- S2安全通過: 暁の衛生兵専用踏査を達成して安全帰還
- S3到達: 現在の遠征でS2を勝利または安全通過
- S3再戦省略: 灰角大猪撃破を安全帰還で確定済み
- 実効RF位置: S1勝利後。S2後はC1、S3後は通常の地域帰還を使う
- 帰還信号技術は次地域で解放されるため、初回攻略時は帰還信号弾UIを表示しない。解放後の再訪だけ有効

### 2. 沈黙した監視所群

```text
入口 -> S1 シンダーウォッチ外門 [RF]
     -> S2 灰道の監視所 -> C1
     -> J1
        +-> S3 アイアンウォッチ物資庫 [RF] -+
        +-> S4 旧兵舎 [RF] ------------------+-> J2 -> C2
     -> S5 信号塔下層 [RF]
     -> S6 最後の信号
     -> 地域出口
```

- S3とS4は任意順。片方の勝利後にC1へ戻って帰還可能
- C2恒久利用とS5到達: S3・S4両方を安全帰還済み
- S1安全通過: 記録箱回収と勝利
- S2安全通過: 監視地点2か所確保
- S3省略効果: 工兵救出と工具区画確保で補給地点化。Node通過自体は維持
- S4安全通過: 伝令路確保と防衛成功
- S5軽量化: 制御盤2個保全でS6の増援公開。S5自体の完全省略はしない
- S6到達: S3・S4確保、S5勝利、主信号経路が通行可能
- 実効RF位置: S1、S3、S4、S5の勝利後。S2後とJ2後はCampを使う

- S3は一方のVariant確保で勝利可能。両Variant確保でS4・S5を軽量化

- S3省略: 旧採掘坑補強と巻上機復旧の両方を確定済み。片方だけなら未確保側を再選択可能
- S4安全通過: 安全採掘手順を安全帰還で確定
- S5到達: S3の最低1Variant確保、S4勝利または安全採取
- 実効RF位置: S1、S3の勝利後。S2・S4後はCamp、S5後は地域帰還

## 帰還信号弾の共通規則

- 使用可能: 戦闘勝利後の地点間画面、次地点の探索選択前
- 使用不可: 戦闘中、探索結果確定後、敗北確定後、強制追跡中、報酬確定処理中
- Campでは無料の「拠点へ帰還」を優先し、帰還信号弾を消費しない
- 地域最終地点勝利後は通常の地域帰還を使い、帰還信号弾を表示しない
- 分岐画面では次の枝を選ぶ前だけ使用できる
- 使用確認には確定するPending 4分類と消費する信号弾を表示する
- 使用確定後に安全帰還Transactionが失敗した場合、信号弾を減らさず同じ画面で再試行する

## 安全通過と省略の不変条件

1. 経路成果は安全帰還後だけ恒久化する。
2. 同じ遠征中の勝利は先へ進む条件になるが、敗北後の次遠征には残らない。
3. 安全通過でHP、Item、Skill回数を消費しない。
4. 補給・回復効果を持つNodeは完全削除せず、非戦闘Nodeとして処理する。
5. 分岐合流のAND条件を安全通過で迂回しない。
6. 兵種または探索道具がなくても主経路を攻略できる。
7. 最終地点の初回攻略を安全通過できない。
8. 地域攻略後の再訪ではBoss・強敵を強制再出現させない。
9. 省略しても地域Map上の通過履歴を残し、入口から進んだことを表示する。
10. Save復元時は解決済みNode、現在Node、選択済みBranch、利用済みCamp効果を復元する。
11. 地域入口から連続する経路確保済みNodeは一括通過でき、地点ごとの確認操作を要求しない。
12. 一括通過は分岐、未解決Node、条件未達Node、既知のCampで停止する。
13. 既知のCampでは通過継続、Camp利用、拠点帰還を選択できる。
14. 安全通過ではHP回復、戦闘不能復帰、補充、報酬、勝利数、ランダムイベントを発生させない。
15. 再調査報酬は地点別`reconLoot`だけとし、初回報酬、Discovery、重要素材、加入、地域進行を含めない。

## データ検証

- Regionごとの`Site`数が`3,6,5,7,6,5,8,6,7,9`で合計62
- 入口から全Siteと出口へ到達可能
- 条件未達成時に後半・最終地点へ到達不能
- 分岐の片方から帰還でき、両方確保後に合流できる
- 各Campが指定地点の後に1回だけ現れる
- 帰還信号弾が戦闘中と無料帰還可能Campで消費されない
- 安全通過で初回報酬とDiscoveryが再付与されない
- どの恒久成果の組合せでも進行不能の閉路を作らない
- 地図外縁S9は敵全滅だけで勝利せず、帰還Objectiveを要求する
