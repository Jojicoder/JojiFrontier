# キャラクター成長仕様

状態: 全12人の横方向成長、加入時装備、連携作戦、調整特性、施設研究費、ユニットページの正本。
共有世界の軍制・命名はWorld Bible、兵種の役割は[`class_reference.md`](class_reference.md)、
個別人物像は[`cast_reference.md`](cast_reference.md)を優先する。

参照したWorld Bible:

- `../../JojiWorldBible/docs/world/embermarch.md`
- `../../JojiWorldBible/military.md`
- `../../JojiWorldBible/naming.md`
- `../../JojiWorldBible/characters.md`

## 成長の原則

キャラクターは経験値、レベル、恒久能力値上昇では成長しない。同一人物の兵種変更も採用しない。

```text
加入
-> 兵種固有能力とTier 1スキル
-> 地域Discoveryと訓練所でTier 2・3スキル
-> 鍛冶場で武器分岐と調整特性
-> 交流区画の会話で連携作戦
-> 遠征前に武器1、特性1、スキル2、連携作戦1を選ぶ
```

強さは同じ行動の数値を上げ続けることではなく、敵・任務・地形に合う構成を選べることから生まれる。

## キャラクター固有能力の境界

- 個人専用の恒常戦闘パッシブは追加しない
- 基礎能力値は人物ではなく兵種から得る
- 同じ兵種の人物が将来加入しても、同一条件なら同じ戦闘性能を再現できる
- 人物差は会話、探索時の反応、連携作戦、初期装備構成へ置く
- 連携作戦は両者を編成する代償があり、単独人物を必須枠にしない

## 連携作戦

交流区画で対応する会話2件を読み、指定地域成果を安全帰還させると解放する。遠征前に部隊全体で
1つだけ選択でき、対応する2人を同時編成した戦闘で1回だけ使用可能。発動者の行動を終了し、
相方は行動済みにならない。戦闘終了で回数を回復する。

| ID | 対応人物 | 名前 | 効果 | 解放条件 |
|---|---|---|---|---|
| `paired_fallback_line` | ライダン、マエラ | 帰還線 | 両者と隣接味方のDEF+2。次のEnemy Phase終了まで | 沈黙した監視所群攻略、指揮と撤退の会話2件 |
| `paired_cross_observation` | ネリス、セラ | 交差観測 | 敵1体を標的指定し、その敵の次回行動候補を同時公開 | 風裂き高原の風見台確保、観測会話2件 |
| `paired_field_recovery` | トーヴァン、ヴェイラ | 野外救護 | 距離2以内の味方1人を8回復し、毒か炎上のどちらか1つを解除 | 黒水低湿地の薬草洲確保、医療会話2件 |
| `paired_braced_breakthrough` | コレン、ハドリク | 支え合う突破 | 両者へ強制移動無効と、突撃してきた敵に対するDEF+2 | 灰鉄採石場攻略、前衛会話2件 |
| `paired_rapid_works` | オレン、カエル | 迅速工作 | 距離2以内の空きマスへ耐久6の臨時防護板を置き、カエルは通常の再移動だけ行える | 風裂き高原攻略、輸送会話2件 |
| `paired_signal_ward` | レッサ、イリエン | 灯火の結界 | 距離2以内の味方のRES+2。次のEnemy Phase終了まで | 埋没聖堂攻略、信号と魔法の会話2件 |

- 連携作戦は装備スキル2枠を消費しない
- 両者が生存し、発動者から相方まで距離2以内の場合だけ使用可能
- 再行動、蘇生、完全無敵、恒久地形変更は与えない
- 同名強化と`skill_system.md`のDEF・RES・MOV上限に従う
- 連携作戦を装備していなくても、全主目的を達成可能にする

## 調整特性

調整特性は製作済み武器へ1つ装着する小規模変更。特性自体は一度製作すれば恒久在庫となり、
別武器へ付け替え可能。遠征中は変更できない。同一特性を同時に複数武器へ装着する場合は、必要数を
個別製作する。

| ID | 日本語名 | 効果 | 制限 | 費用 | 解放 |
|---|---|---|---|---|---|
| `hide_wrapped_grip` | 獣皮の柄巻き | 戦闘ごとに最初に受けるノックバックを1回無効 | 全武器 | 獣皮1 | 灰枝の森攻略 |
| `ashwood_balance` | 灰木の重心調整 | その行動で1マス以下しか移動していない通常攻撃の与ダメージ+1 | 杖・焦点具不可 | 木材2、獣皮1 | 森の踏査記録 |
| `field_sling` | 野戦吊り紐 | アイテム使用対象の選択射程を隣接から距離2へ変更 | 威力・回復量は変えない | 織物1、獣皮1 | 野戦工作記録 |
| `shallow_seal` | 浅瀬封じ | 浅瀬で行動終了した次のEnemy PhaseまでRES+1 | 全武器 | 湿地樹脂2、織物1 | 湿地踏査記録 |
| `wind_stable_mount` | 防風固定具 | 強風による最初の強制移動を戦闘ごとに1回無効 | 弓・槍・焦点具 | 硬木1、騎具素材1 | 高原街道図 |
| `heat_guard_wrap` | 耐熱覆い | 攻撃または地形から受ける最初の炎上を戦闘ごとに1回無効 | 全武器 | 耐熱素材2、湿地樹脂1 | 耐熱加工記録 |
| `reinforced_socket` | 補強石突き | 障害物へ与えるダメージ+2 | 槍・大槌・工作槌 | 鉄材2、木材1 | 重装加工記録 |
| `sanctum_binding` | 聖堂式留め具 | 使用者が行う非アイテム回復+1 | 杖のみ | 聖堂器材1、薬草2 | 医療典籍 |

調整特性の効果は小さく保ち、武器分岐の代償を消さない。MOV低下、射程低下、威力低下を特性で
相殺しない。命中率、必殺率、恒久能力値は変更しない。

## 施設研究費

全施設研究ノードの安定ID、効果、Discovery、素材、前提研究、解放地域、種別は
[`facility_research.md`](facility_research.md)を唯一の正本とする。このファイルでは個別武器、
調整特性、連携作戦など、ユニットへ装備する成果物の費用だけを定義する。

研究費を払った結果、次の必須拠点発展が不可能になっても進行不能にはしない。確保済み地点から
通常素材を再取得できる。UIは次の拠点発展で予約すべき素材数を同時表示する。

## ユニットページ

### 一覧

- 左側に加入済みUnitを表示し、名前、兵種、現在HP、戦闘不能、遠征編成中を示す
- 未加入者の名前と兵種は表示せず、加入候補を発見済みなら「加入候補あり」とだけ表示する
- 比較対象を1人固定できるが、能力値差を緑赤だけで表現しない

### 詳細

1. 名前、兵種、兵種役割、固有能力
2. HP、STR、MAG、DEF、RES、MOV
3. 武器1枠と武器固有効果
4. 調整特性1枠
5. 装備スキル2枠と残り方式
6. 選択中の連携作戦と必要な相方
7. 探索能力

武器、特性、スキルを選ぶと、右側へ「現在」「変更後」「変わる戦術」「失うもの」を表示する。
長文は折り返し、ホバーだけに必須情報を置かない。

### 変更規則

- 拠点かつ遠征開始前だけ変更可能
- 武器変更は製作済み武器だけ
- 所有済み調整特性と製作済み武器はいつでも拠点で変更可能
- 解放済みスキルはいつでも拠点で変更可能
- 解放済み連携作戦は交流区画解放後、施設状態にかかわらず変更可能
- 新規製作・研究は対応施設の建設を必要とする
- 無効な組合せは理由と不足条件を日本語で表示する
- 確定前に保存せず、「変更を確定」でロードアウトを原子的に更新する

## セーブデータ

```cpp
struct UnitLoadout {
    UnitId unitId;
    WeaponId weaponId;
    std::optional<TuningTraitId> tuningTraitId;
    std::array<std::optional<SkillId>, 2> equippedSkills;
};

struct PartyTactics {
    std::optional<CooperationId> equippedCooperationId;
};
```

保存対象は製作済み武器・特性数、解放済みスキル、解放済み連携作戦、Unit別ロードアウト、
部隊の連携作戦1枠。戦闘内の残り回数、クールダウン、一時効果は保存しない。

## バランス受入条件

- 基本武器、Tier 1スキル、特性なしでも全主目的を達成可能
- 上位装備は副目標と安全性を改善するが、敵能力値を無視できない
- 連携作戦あり・なしの平均被害差は5～15%を目安とし、必須化しない
- 1つのロードアウトが全地域で最適にならない
- 人物固有差によって同兵種の代替人物が明確な下位互換にならない
- 装備変更前後の攻撃範囲、MOV、回復量、状態付与をプレビューへ反映する
