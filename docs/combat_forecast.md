# 攻撃予測・危険予告表示仕様
## 状態

共通UI・情報公開仕様として確定。予測画面はCombat Resolverと同じ入力から結果を算出するが、
BattleState、使用回数、乱数、Objectiveを変更しない。

## 基本原則

- 行動確定前に、プレイヤーが受け入れる損失と位置変化を確認できる
- 予測専用の簡易ダメージ式を作らず、実処理と同じ純粋計算関数を使う
- 未公開情報を表示しないが、非公開情報を理由に即死級の結果を隠さない
- Seed、内部ID、AI Scoreは通常UIへ表示しない
- 色だけで範囲の意味を区別しない

## 通常攻撃予測

対象選択中に次を表示する。

- 攻撃者名、武器、攻撃名
- 対象名、現在HP、予測後HP
- ダメージ値と物理・魔法区分
- 命中率。通常100%、茂みなど明示された回避補正後の実値
- 付与予定の状態異常とBoss補正後の効果
- ノックバック方向、移動先、衝突ダメージ
- Objectへの耐久ダメージと破壊後状態
- 発動可能な反撃準備などの反応
- 撃破、Objective進行、味方敗北条件に直接関係する警告

通常反撃は存在しないため、常設の反撃ダメージ欄を置かない。反応条件が成立する場合だけ
「反応: 反撃準備」と予測ダメージを表示する。

## 命中と不確定結果

- 基本必中攻撃は`命中 100%`
- 茂み上の防御側など回避が存在する場合は実際の命中率を表示
- 複数の結果がある場合は命中時と回避時を分け、平均ダメージだけを表示しない
- 乱数結果そのものは確定前に公開しない
- 状態異常は命中すれば確定付与。別の付与確率を作らない
- 完全無効は`無効`、軽減は補正後数値を表示する

## 範囲攻撃

- 対象マスと影響全マスを盤面へ表示
- 敵、味方、Objectごとに予測結果を一覧化
- 同一Unitが複数占有マスに触れても1回分だけ表示
- 味方を巻き込む場合は確定前に警告
- 中心変更で一覧を即時再計算するが、レイアウトを動かさない
- Boss範囲攻撃へ通常反撃欄を表示しない

## 危険予告の種類

| 表示 | 用途 | 見た目 |
|---|---|---|
| 移動可能 | 味方の停止可能範囲 | 青面＋移動アイコン |
| 攻撃可能 | 選択Unitの武器射程 | 薄赤面＋照準アイコン |
| 攻撃対象 | 実際に対象がいるマス | 濃赤面＋対象枠 |
| Boss予告 | 次回固定攻撃 | 赤縞＋行動アイコン＋残りRound |
| 増援予告 | 出現候補 | 黄枠＋入口アイコン＋残りRound |
| 地形変化 | 崩落、炎上、噴気 | 橙点滅枠＋地形アイコン |
| Objective | 確保、保全、脱出 | 金枠＋目的アイコン |

既存の青い移動範囲と赤い攻撃範囲を維持する。紫を意味未定義の共通色として使用しない。

## 重複表示

同じマスへ複数予告が重なる場合:

1. 敗北条件に直結する危険
2. Boss予告
3. 地形変化
4. 増援
5. 通常攻撃範囲
6. Objective

最優先の面表示を使い、右上に最大3個の小アイコンを並べる。4件以上は`+N`とTooltipで展開する。
点滅は実行直前の重大予告だけに限定し、複数速度で点滅させない。

## 敵危険範囲

- 敵選択時は現在位置からの移動可能範囲と攻撃可能範囲を表示
- 全敵危険範囲の一括表示はToggleで切り替える
- 未行動敵と行動済み敵を線種で区別する
- 増援は出現Phaseに行動しないため、出現予告と攻撃危険範囲を同時表示しない
- 予告済みBoss行動は通常AIの推定危険範囲より優先する
- AIが選ぶ実対象は予告固定行動以外では公開しない

## 情報公開レベル

| 情報 | 通常 | 偵察済み |
|---|---|---|
| 盤面上の敵HP・武器射程 | 公開 | 公開 |
| 通常攻撃予測 | 公開 | 公開 |
| 増援方向・残りRound | 公開 | 公開 |
| 増援兵種・正確な候補順 | 非公開可 | 公開 |
| Boss予告範囲と効果 | 公開 | 公開 |
| Boss次回AI候補 | 非公開 | 猟兵など固有効果時のみ公開 |
| 隠し副目標・未知Object | 非公開 | 発見能力に応じて公開 |

「敵影あり」が勝利を止める場合は必ず表示する。偵察不足を理由に予告範囲や敗北条件を隠さない。

## データ契約

```cpp
struct CombatForecast {
    ActionDefinitionId actionId;
    std::vector<ForecastTargetResult> targets;
    std::vector<GridPos> affectedTiles;
    std::vector<ForecastWarning> warnings;
    std::optional<ReactionForecast> reaction;
};
```

Forecast計算は`const BattleState&`を受け、状態変更Eventを発行しない。実行時は同じAction Definitionと
確定対象をResolverへ渡す。予測と実行の差が乱数以外で生じた場合はテスト失敗とする。

## メッセージボックスとの分担

- 予測Panelは現在カーソル中の行動だけを表示
- 盤面Overlayは位置と範囲を表示
- メッセージボックスは確定済みEvent、Boss・増援予告、Objective変化を履歴表示
- 同じ予測文をメッセージ履歴へ追加しない
- 行動確定後は予測を閉じ、結果の短い数値Feedbackだけ表示する

## 受入条件

- 通常攻撃の予測ダメージと実ダメージが一致する
- 茂み回避時は100%ではなく補正後命中率を表示する
- ノックバック先、衝突、状態異常、反撃準備を確定前に表示する
- Boss範囲攻撃に反撃表示を出さない
- 増援出現Phaseの即時攻撃範囲を表示しない
- 複数予告が重なっても各意味をTooltipで確認できる
- 未公開内部ID、Seed、AI Scoreを表示しない
- Forecast計算でBattleStateが変化しない
