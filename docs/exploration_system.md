# JOJIFrontier 探索システム

状態: JOJIFrontier固有のゲームシステム案。地域名、人物、歴史へ接続する場合は
`../JojiWorldBible` を確認する。

## 基本方針

探索に自由移動マップは使用しない。遠征ルート上のノードを進み、各探索ノードで
状況文と選択肢を提示する。単なるノベル分岐ではなく、選択結果を直後の戦闘、消耗、
戦利品、発見物へ反映する。

基本画面遷移:

```text
BASE
-> ROUTE NODE
-> EXPLORATION EVENT
-> RESULT
-> BATTLE
-> LOOT
-> CAMP
-> NEXT NODE または BASE
```

基本ループ:

```text
拠点
-> 探索ノード（状況文＋選択肢）
-> 戦闘
-> 戦利品
-> キャンプ（続行／帰還）
-> 次の探索ノード
```

## 選択肢の構成

テスト版では各地点に次の3択を用意する。

- 通常選択肢2つ
- 編成中の兵種によって使用可能になる専用選択肢1つ

兵種専用選択肢は常に最善にはしない。安全性、情報、戦利品、消耗、時間などの
トレードオフを持たせる。選択前には「安全」「危険」「物資消費」「戦闘有利」など、
結果の性質をある程度予告する。

## 選択結果が変更できる要素

- 敵の数と種類
- 敵味方の初期配置
- 地形と障害物
- 戦闘開始時HP
- アイテム消費
- 追加戦利品
- 施設解放に使う発見物
- 戦闘回避
- ボス能力や増援条件

戦闘回避を常に最適解にしない。戦闘を避けた場合は戦利品、発見物、地域確保などを
得られない場合がある。

## 兵種による探索能力

| 兵種系統 | 探索能力 |
|---|---|
| Frontier Scout | 偵察、隠し道、高所からの観測 |
| Frontier Engineer | 障害物、機械、橋、遺跡装置 |
| Frontier Ranger | 追跡、狩猟、野生生物への対応 |
| Dawn Chirurgeon | 負傷者、医療記録、衛生上の危険 |
| Heavy Infantry | 瓦礫除去、強行突破、重量物運搬 |

兵種の価値を戦闘能力だけでなく、遠征ルートと資源管理にも持たせる。

## 最初の3ノード

### Cinderwatch Gate

判断テーマ: 情報を得るか、急いで進むか。

状況:

> 崩れた門の先に、放棄された監視所が見える。正面の通路には新しい足跡が
> 残っている。

確定したテスト仕様:

#### A. 正面突破 / Frontal Advance

条件なし。基準ルート。

- 味方HP変化なし
- 敵編成は通常
- 敵配置は非公開
- 味方初期配置は通常
- 戦闘勝利で重要資料を取得可能

#### B. 崩れた側道を進む / Take the Collapsed Side Path

条件なし。事前消耗と引き換えに戦闘を軽くするルート。

- 生存中の味方全員に2ダメージ
- 敵1体を戦闘開始時から除外
- 敵配置を一部公開
- 味方初期配置は通常
- 戦闘勝利で重要資料を取得可能

初期バランス値は `HP -2 x 4人 / 敵 -1` とする。敵1体除外が強すぎる場合は、
全員3ダメージ、最弱敵のみ除外、通常戦利品減少の順に調整候補とする。

#### C. `[Frontier Scout]` 高所から偵察 / Scout from High Ground

Frontier Scoutを遠征隊に編成している場合のみ選択可能。敵数を減らさず、情報と
初期配置の優位を得る。

- HP消耗なし
- 敵数と敵編成は通常
- 敵編成を全公開
- 敵初期配置を全公開
- 味方4人を戦闘開始前に自由配置
- 戦闘勝利で重要資料を取得可能

自由配置の制限:

- 左3列以内
- 同一マス不可
- 通行可能地形のみ
- 敵陣への直接配置不可

#### 重要資料

この地点の重要資料は「偵察資料 / Reconnaissance Records」とする。

```text
探索選択
-> Cinderwatch Gateの戦闘に勝利
-> PendingDiscoveriesへ偵察資料を追加
-> 安全に拠点へ帰還
-> Discovery Registryへ恒久登録
-> 司令所「偵察網」を解放可能
```

探索選択だけでは取得できない。戦闘を回避した場合は重要資料も地域確保も得られない。

### Ironwatch Stores

判断テーマ: 物資を守るか、消耗を受け入れて確保を急ぐか。

選択肢の役割:

- 慎重に倉庫へ入る: 標準戦闘、物資を保全
- 崩れた搬入口を突破する: パーティに軽い損傷、敵数または地形が有利
- `[Dawn Chirurgeon / Heavy Infantry]` 医療区画を調査、または瓦礫を除去する:
  追加資料や物資を得る代わりに別の危険を引き受ける

主な戦闘変更: HP、敵数、追加戦利品。

報酬候補: 野戦医療資料。安全帰還後、診療所「野戦医療」の条件になる。

### Signal Tower

判断テーマ: 安全を確保するか、ボス戦を有利にする工作を優先するか。

選択肢の役割:

- 正面階段を確保する: 標準ボス戦
- 外周設備を停止する: 道中消耗と引き換えに増援を抑制
- `[Frontier Engineer]` 信号装置を逆利用する: 障害物、ボス能力、初期配置のいずれかを変更

主な戦闘変更: 障害物、ボス能力、増援条件。

報酬候補: 信号技術資料。安全帰還後、工房「帰還信号技術」の条件になる。

## 結果データ

探索結果は共通データへ変換し、イベント固有処理を戦闘コードへ直接書かない。

```cpp
struct ExplorationOutcome {
    int partyDamage = 0;
    std::vector<ItemType> consumedItems;
    std::vector<std::string> pendingLoot;
    std::vector<std::string> pendingDiscoveries;
    BattleModifier battleModifier;
    bool skipBattle = false;
};
```

`BattleModifier` は敵数、初期配置、地形、ボス能力、増援条件などを保持する。
探索結果画面と戦闘HUDでは、適用された効果を明示する。

例:

- 「斥候成功: 敵編成を確認」
- 「有利な初期配置」
- 「工兵工作: Barrierを除去」
- 「信号停止: 増援なし」

## 戦利品と発見物の分離

通常戦利品と進行用の重要発見は内部データを分離する。

`PendingLoot`:

- 鉄材
- 武器
- 消耗品
- 通常素材

`PendingDiscoveries`:

- 偵察資料
- 野戦医療記録
- 信号技術資料
- 地域ボス由来の重要発見

安全帰還時の処理:

```text
PendingLoot
-> 倉庫・所持品へ確定

PendingDiscoveries
-> Discovery Registryへ恒久登録
-> 対応する拠点ノードを解放可能にする
```

重要発見は売却、装備、消費の対象にしない。敗北時には通常戦利品と同様、まだ帰還して
いないPending Discoveriesを失う。

## 日本語・英語対応

探索システムの全表示は日本語と英語の両方に対応する。

- 地点名
- 状況文
- 選択肢名と説明
- 条件タグと選択不可理由
- リスク／効果の予告
- 探索結果
- 戦闘へ適用された補正
- 戦利品名と発見物名

イベントロジックでは表示文字列を判定キーに使わない。イベント、選択肢、発見物には
言語非依存の安定IDを持たせ、表示文だけを言語データから取得する。

例:

```text
choice id: cinderwatch_high_ground_scout
ja: 【斥候】高所から偵察
en: [Frontier Scout] Scout from high ground
```

未翻訳時に別言語へ暗黙フォールバックして仕様不足を隠さず、開発時に欠落を検出できる
ようにする。

## 実装順

1. Exploration Event、Choice、Outcomeのデータモデル
2. PendingLootとPendingDiscoveriesの分離
3. 日英対応の探索画面と結果画面
4. Cinderwatch GateのA/B/C選択
5. Bの事前ダメージ、敵1体除外、一部情報公開
6. Cの完全情報公開と自由初期配置画面
7. 戦闘勝利で偵察資料をPendingDiscoveriesへ追加
8. 帰還時にDiscovery Registryへ登録
9. Ironwatch StoresとSignal Towerへ展開

最初は複雑な分岐ルートやランダムイベントを実装しない。3地点を直線で接続し、
「誰を編成したかが探索と戦闘の両方を変えるか」を検証する。
